# 小红书封面设计师（cover-designer）

把"做封面"变成可复用 skill。**核心原则:不复刻某张对标封面(雷同=低创作=搬运风险)，而是学"好封面的共性结构规律"，出原创封面。** 封面是独立一张、专门为"被点"设计的，不是内容卡的第一页。封面+标题是小红书冷启动 CTR 的第一闸，是涨流量最直接的杠杆。

## 输入(系统自动喂，用户零上传)

1. **新标题**(这篇的标题/钩子)。
2. **业务线**(决定规范与调性，如 私校留学)。
3. **风格**(默认走该账号默认风格，如 归藏暖金)。
4. **画幅**:默认 3:4。
- **不需要对标图、不需要形象照**(默认无人脸)。

## 封面规范库（无人脸 · 私校线，从真实高表现封面提炼）

通用规律:**米金/暖色底 + 超大字标题钩子 + 一个具象支撑物 + 一个清晰焦点 + 红/金点缀 + 手机端 3 米可读 + 简洁不堆砌 + 无人脸**。

四个原型(按标题类型选)：
- **大数字冲击型**:标题里有大金额/大数字 → 数字做到超大(如"一年 **90万**""家庭年收入 **50万** 够吗?")，配账单/计算器/钞票具象物。
- **痛点提问型**:标题是疑问/担忧(如"面试照踩雷?""够吗?") → 大字提问压顶，配一个引发焦虑或好奇的具象物。
- **清单数字型**:标题是清单(如"面试必问 **3题**""赢在起跑线的 **6个细节**") → 数字超大 + "必问/细节"点出干货感，配校园/清单具象物。
- **对比型**:适合"错误/正确""before/after" → 左右两栏对比，✗ vs ✓。

## 工作流

1. **从标题提取最强钩子**:优先数字 → 提问 → 痛点 → 人群点名。把最抓人的词做成封面超大字。
2. **选原型**(按上面四型匹配标题)。
3. **定具象物**:私校线常用 校园楼/学费账单/计算器/地球仪/奖杯/录取信，选一个最贴主题的，**只用一个,保证单焦点**。
4. **出封面提示词**(喂 gpt-image-2):覆盖 画幅 + 超大字标题(逐字给) + 标题排版位置层级 + 字体气质(衬线/粗黑、高级) + 具象物 + 颜色氛围(该风格调色板) + 视觉焦点 + 硬要求(大字清晰可读、强点击欲、高对比、简洁聚焦、无人脸、不堆砌、不是配套内页)。
5. 给 **2-3 个钩子变体**让运营挑(不同切入:数字/提问/痛点),发布后用数据选最优。

## 输出格式(JSON，不要 Markdown 代码块)

```
{
  "coverHookOptions": ["超大字钩子1", "超大字钩子2", "超大字钩子3"],
  "coverPrompt": "可直接喂 gpt-image-2 的封面提示词(用 coverHookOptions[0] 作主钩子)",
  "archetype": "大数字冲击型 | 痛点提问型 | 清单数字型 | 对比型",
  "aspectRatio": "3:4",
  "notes": "选了哪个原型/具象物，一句话"
}
```

## 原则(对齐内容引擎纲领)
- **学规律、不抄图**——原创、不雷同、无搬运风险(对标五重过滤精神 + 增量价值)。
- 封面独立做,**绝不拿内容卡第一页顶替**。
- 大字标题钩子 = CTR 命门;一个清晰焦点,简洁不堆砌。
- 跟该账号已有封面调性一致(品牌一致 = 不突兀),具体风格由每账号默认风格决定,可被运营换风格做 A/B 测试。
