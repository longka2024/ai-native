# Longka 视频成片助理 PRD

## 1. 一句话定位

给小老板、运营、小妹和行业从业者用的 AI 视频成片助理：把文案、素材、案例、群聊需求变成可以发布到视频号、小红书、抖音的短视频草稿。

客户买的不是视频工具，而是结果：

```text
我不会剪视频，也能每天做出能发的内容。
```

## 2. 被私域群聊验证出的真实需求

从微信群 200 条聊天里提炼出的核心需求：

- 领导/老板年纪大、不想学工具，只想一键出想要的结果。
- 用户希望把文案丢进去，直接出来成片，能发视频号。
- 现有工具能做，但流程不稳定：抖动、硬切、素材混乱、需要重做。
- 素材不是越多越好，参考图太多反而让 AI 混乱。
- 小白不知道做什么，需要故事板和固定模板。
- 六宫格故事板比九宫格更适合入门和执行。

## 3. 目标用户

### 第一批用户

- 美业/形象设计从业者
- 家装/本地服务老板
- 小红书/视频号运营小妹
- 个人 IP 内容创作者
- 不懂技术但想用 AI 做内容的小老板

### 他们的共同点

- 有素材，但不会组织。
- 有想法，但不会写脚本。
- 会发平台，但不会稳定生产。
- 不想学复杂工具。
- 希望员工照流程执行。

## 4. 用户真正想点击的按钮

第一版不要暴露 Remotion、RunningHub、即梦、模型、MCP。

界面只放结果型按钮：

```text
1. 文案生成视频
2. 图片生成视频
3. 案例生成视频
4. 群聊生成选题
5. 选题生成脚本
6. 脚本生成分镜
7. 一键生成成片草稿
8. 重新生成某一段
```

## 5. 核心工作流

### 工作流 A：文案到成片

输入：

- 一段文案
- 行业
- 平台：视频号 / 小红书 / 抖音
- 视频时长：15 秒 / 30 秒 / 60 秒
- 风格：干货 / 情绪 / 对比 / 案例 / 口播

流程：

```text
文案
-> 提炼主题
-> 生成钩子
-> 拆成六宫格故事板
-> 每格生成画面描述
-> 匹配用户素材或生成素材
-> 生成字幕
-> 生成剪辑节奏
-> 渲染成片草稿
-> 生成封面和标题
```

技术：

- LLM：文案理解、钩子、故事板、字幕
- imagegen / gpt-image2 / KIE / 即梦 / RunningHub：画面生成
- Remotion：程序化视频渲染
- FFmpeg：后处理、压缩、转码、拼接
- HyperFrames / zine-deck-maker：视觉节奏和版式灵感

### 工作流 B：素材到成片

输入：

- 3-6 张核心图片
- 可选文案
- 行业模板

流程：

```text
素材筛选
-> 判断素材用途：人物/场景/产品/案例/证明
-> 去重和排序
-> 生成故事线
-> 每张图绑定一个信息点
-> 生成视频结构
-> 渲染成片
```

关键规则：

- 参考图不是越多越好。
- 默认 3-4 张最稳。
- 超过 6 张时先让 AI 做素材分组，不直接喂给生成模型。
- 用户上传太多素材时，先让系统选“最适合入片的 5 张”。

### 工作流 C：群聊到内容

输入：

- 微信群最近 N 条聊天

流程：

```text
wx-cli 读取聊天
-> 脱敏
-> 总结高频问题
-> 提炼客户痛点
-> 找购买信号
-> 生成内容选题
-> 生成视频脚本
-> 进入文案到成片流程
```

技术：

- `wx-cli`：本地授权读取微信
- Longka private-domain intelligence：群聊总结
- Longka content creative：选题/脚本
- 视频成片工作流：成片

隐私规则：

- 默认不上传原始聊天。
- 只输出摘要、选题、话术。
- 产品化时必须让用户主动选择群。

### 工作流 D：案例到宣传视频

输入：

- 客户案例
- 前后对比图
- 成品图
- 服务说明

流程：

```text
案例摘要
-> 提炼前后反差
-> 生成标题
-> 生成 15/30 秒宣传脚本
-> 选择大图冲击画面
-> 生成节奏字幕
-> 渲染视频
```

适合：

- 色彩报告
- 美业前后对比
- 家装案例
- 旅游路线
- 律师普法案例，需合规处理

## 6. 技术模块拆解

### 6.1 输入层

来源：

- 手写文案
- 图片素材
- 微信群聊
- 小红书/公众号爆款
- 客户案例
- 产品介绍

技术：

- 文件上传
- wx-cli
- Agent Reach
- Exa / mcporter
- Jina Reader
- 本地文件读取

### 6.2 理解层

任务：

- 判断行业
- 判断目标用户
- 判断平台
- 提炼痛点
- 提炼钩子
- 判断素材质量

技术：

- LLM
- 行业模板
- Longka skill matrix
- 内容情报 SOP

### 6.3 结构层

任务：

- 六宫格故事板
- 镜头节奏
- CTA
- 字幕结构
- 封面句

技术：

- `longka-content-creative`
- 新增 `longka-video-workflow-assistant`
- Remotion composition schema

### 6.4 素材层

任务：

- 选择图片
- 去重
- 排序
- 生成缺失画面
- 保持人物/产品一致

技术：

- imagegen
- gpt-image2
- KIE / Flux Kontext
- RunningHub / ComfyUI，后续可接
- 本地素材库

### 6.4.1 可插拔生成引擎

视频成片助理不能绑死在一个模型或一个平台。生成层要做成 Provider 模式：

```text
ImageProvider:
  - gpt-image2
  - KIE / Flux Kontext
  - ComfyUI
  - RunningHub
  - 即梦

VideoProvider:
  - Remotion 本地渲染
  - ComfyUI 视频工作流
  - RunningHub 视频工作流
  - Seedance / Sora，后续可接

AvatarProvider:
  - 数字人克隆
  - 口播驱动
  - 声音克隆
```

客户不需要知道 provider。客户只看到：

```text
生成形象图
生成视频草稿
生成数字人口播
重新生成这一段
```

### 6.4.2 ComfyUI 工作流位置

ComfyUI 适合承担：

- 固定风格图像生成
- 人物一致性工作流
- 产品图/案例图增强
- 数字人形象生成
- 图生视频
- 局部重绘
- 批量生产

接入方式：

```text
Longka 前台按钮
-> 工作流参数表单
-> ComfyUI workflow JSON
-> 队列执行
-> 产物回收
-> 进入 Remotion/FFmpeg 合成
```

### 6.4.3 数字人克隆

数字人克隆属于高级能力，不放在 MVP 第一屏。

适合场景：

- 老板不想真人出镜，但想做口播。
- 律师/老师/顾问做知识短视频。
- 美业/形象设计做讲解型内容。
- 家装老板做避坑讲解。

产品按钮：

```text
生成我的数字人口播
用我的声音读这段文案
把这段脚本做成口播视频
```

合规边界：

- 必须本人授权。
- 不做冒充他人。
- 不默认克隆客户聊天里的任何人。
- 输出需保留“AI 生成/数字人”提示策略，按平台和当地规则处理。

### 6.5 渲染层

任务：

- 视频布局
- 字幕
- 转场
- 节奏
- 片头片尾
- 导出尺寸

技术：

- Remotion
- FFmpeg
- HyperFrames
- Anime.js 动效灵感

### 6.6 质检层

任务：

- 脸有没有被裁掉
- 字幕有没有挡住主体
- 是否硬切太明显
- 是否节奏太快/太慢
- 图片是否抖动
- 是否符合平台比例

技术：

- Playwright 截图
- FFmpeg 抽帧
- 简单视觉规则
- 人工确认

### 6.7 交付层

输出：

- MP4
- 封面图
- 标题
- 发布文案
- 评论区引导话术
- 朋友圈文案
- 可修改的脚本

## 7. 第一版 MVP

先做最小可卖版本，不做全自动大平台。

### MVP 功能

```text
1. 输入一段文案
2. 选择行业：美业/形象设计
3. 选择平台：视频号/小红书
4. 自动生成六宫格故事板
5. 自动生成视频脚本
6. 用现有素材/样片生成 30 秒视频草稿
7. 输出标题、封面句、发布文案
```

### MVP 不做

- 不做自动发布
- 不做复杂多账号矩阵
- 不做完全无人审核
- 不做所有行业
- 不承诺一次生成完美成片

### MVP 成功标准

- 小妹能照流程做出视频。
- 一个普通老板看了知道点哪里。
- 30 分钟内能从文案得到可用草稿。
- 生成内容比手工从零开始省 70% 时间。

## 8. 风险

- 生成视频不稳定。
- 素材太乱导致画面混乱。
- 客户期望“一键完美”，实际需要二次确认。
- 云端渲染成本不可控。
- API/token 成本失控。
- 微信聊天隐私敏感。
- 不同行业模板质量差异大。
- 数字人克隆涉及肖像、声音和授权风险，必须做授权和用途边界。
- ComfyUI 工作流强但复杂，需要封装成结果按钮，不能让小白直接面对节点图。
- U 盘产品存在复制、盗版、暴力破解和模板泄露风险，必须采用“本地入口 + 云端授权 + 行业包远程下发”的结构。

## 8.1 安全与授权

视频成片助理作为 U 盘产品时，不能把全部核心资产放在本地。

本地可放：

- 启动器
- UI 面板
- 示例模板
- 开源依赖安装脚本
- 本地缓存

云端保留：

- 激活授权
- 核心行业模板
- 高价值 prompt
- token gateway
- 视频渲染队列
- 行业包更新
- 用量统计

## 11. RPA / Computer Use 执行层

视频成片助理后续要接入日工作流执行层：

```text
选题
-> 生成
-> 质检
-> 发布准备
-> 人工确认发布
-> 评论/私信跟进
```

执行技术：

- API / CLI 优先。
- Playwright / RPA 处理网页操作。
- Computer Use 处理复杂桌面和视觉操作。

边界：

- 发布、删除、付款、发送敏感消息必须人工确认。
- 自动化先负责准备草稿和素材，不绕过平台规则。
