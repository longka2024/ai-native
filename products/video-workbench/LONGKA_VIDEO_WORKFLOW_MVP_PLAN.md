# Longka 视频成片助理 MVP 计划

## 1. MVP 目标

用现有色彩报告项目素材，做出第一条可展示的 30 秒竖屏视频：

```text
输入一段卖点文案
-> 生成六宫格故事板
-> 复用报告成品图
-> Remotion 渲染 30 秒视频
-> 输出标题、封面句、发布文案
```

## 2. 为什么先用色彩报告项目

- 已有真实成品图。
- 已有女性用户场景。
- 已有付款/试看/样片流程。
- 已有 Remotion promo-video 项目。
- 可直接验证“客户要结果，不要技术”。

## 3. 第一条视频主题

主题：

```text
你穿衣显土，可能不是脸的问题，是颜色错了。
```

目标用户：

```text
不知道自己适合什么颜色、妆发和穿搭方向的普通女生。
```

CTA：

```text
上传照片，生成你的试看图。
```

## 4. 输入素材

优先使用：

```text
E:\Codex\personal-image-report-product\promo-video\public\report-live
```

需要：

- 1 张素颜/原始参考图，若可用
- 4-6 张报告成品图
- 1 张样片或预览图
- 品牌/小程序名称

## 5. 输出文件

建议输出：

```text
E:\Codex\personal-image-report-product\promo-video\out\longka-video-assistant-mvp-30s.mp4
```

同时输出：

```text
E:\Codex\personal-image-report-product\promo-video\out\longka-video-assistant-mvp-brief.md
```

brief 包含：

- 标题
- 封面句
- 发布文案
- 六宫格故事板
- 使用素材
- 质量检查

## 6. 验收标准

- 30 秒内能讲清楚产品结果。
- 前 3 秒有钩子。
- 中段有大图冲击。
- 不讲技术，不讲模型。
- 成品图不裁脸。
- 字幕不遮挡主体。
- CTA 清楚。
- 小妹看了知道这是给谁看的、要用户做什么。

## 7. 不做事项

- 不做自动发布。
- 不做复杂后台。
- 不做多行业模板。
- 不做完全自动素材生成。
- 不追求一次达到最终商业片，只验证工作流。

## 8. 下一步执行

1. 读取 promo-video 项目结构。
2. 选素材。
3. 生成六宫格故事板。
4. 增加 Remotion composition。
5. 渲染 30 秒视频。
6. 截图/抽帧检查。
