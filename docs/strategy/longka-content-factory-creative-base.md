# Longka 内容工厂创作基座 v1

Longka 不让 AI 直接写文章，而是让 AI 按“选题、拆解、问题、标题、框架、正文、体检、包装”的创作流程逐步工作。

这个文档是 AI Native 内容生产系统的创作基座。后续页面、接口、提示词、测试和小妹视频工作台交接，都必须按这里的步骤验收。

## 核心原则

1. 选题定生死，文案决定爆款程度。
2. 爆款不是照抄句子，而是拆情绪、结构、信任证据和行动入口。
3. 评论区问题不是附属材料，而是重要选题入口和长期资产；但单篇创作不必须依赖某一条评论。
4. 标题不是随机生成，必须能追溯到标题公式和用户问题。
5. 正文不能跳过框架直接生成。
6. 文案确认前，不生成图片、视频、打包、上线或修改生产配置。
7. 内容资产必须沉淀到客户自己的本地资料库，方便长期复用和迁移。

## 七步闭环

### 1. 爆款样本池

输入：

- 行业
- 业务目标
- 关键词
- 参考平台

输出：

- 5-10 条真实采集或手动导入的高赞素材
- 每条必须包含标题、平台、关键词、赞藏评转、评论状态、原帖链接或 traceId、入选理由

入选依据：

- 高赞、高藏、高评优先
- 评论多优先
- 和当前关键词匹配优先
- 有原帖链接优先
- 有评论区问题优先

禁止：

- 不生成正文
- 不生成标题候选
- 不生成图片
- 不生成视频
- 不打包
- 不用旧行业数据或固定模板冒充真实样本

样本不足 5 条时，页面必须明确提示素材不足。

### 2. 样本拆解

只对用户选中的一条源头帖拆解。

输出字段：

- 这条为什么火
- 目标人群
- 表层话题
- 真实情绪
- 隐藏需求
- 用户害怕什么
- 评论区主问题
- 开头钩子
- 信任证据
- 内容结构
- 封面套路
- 可复制结构
- 不能复制什么
- Longka 二创方向

验收标准：

- 每个判断都要绑定源头帖、互动数据或评论问题
- 不能只总结标题和正文
- 不能把评论区当装饰

### 3. 客户问题库

从评论区和源头帖中提炼客户问题。

问题类型：

- 疑问句
- 求助句
- 担忧句
- 购买犹豫句
- 反复踩坑句
- 对比选择句

每条问题必须包含：

- 问题文本
- 来源原帖
- 评论原文
- 赞藏评转
- 关键词
- 适合转成什么选题

这些问题必须成为后续标题和正文的主问题。

### 4. 标题公式匹配

基于选中的源头帖和客户问题，生成 6-10 个标题。

每个标题必须显示：

- 标题
- 公式类型
- 适合人群
- 主问题
- 对应评论问题
- 为什么适合

规则：

- 标题必须覆盖至少 3 种不同触发类型
- 换标题必须换主问题、结构和正文
- 不允许只替换标题文本
- 不允许无来源随机生成标题

### 5. 文案框架生成

正文生成前，必须先展示框架。

框架字段：

- 选中标题
- 主问题
- 目标人群
- 开头策略
- 情绪路线
- 正文结构
- 信任证据
- 评论问题如何进入正文
- 行动入口
- 合规边界
- 配图方向

验收标准：

- 用户确认框架后才写正文
- 框架必须绑定第 4 步选中的标题
- 框架必须绑定第 2 步源头帖和第 3 步客户问题

### 6. 正文生成 + 体检进化

按框架生成小红书图文、朋友圈文案、公众号文章或视频脚本。

生成后必须体检。

Longka 文案体检维度：

- 标题钩子
- 开头留存
- 情绪命中
- 具体性
- 收藏价值
- 转化入口
- 合规边界
- AI 痕迹
- 文字节奏
- 真实感
- 精炼度

Humanizer 子评分：

- 直接性
- 节奏
- 信任度
- 真实感
- 精炼度

必须识别的问题：

- 空话
- 套话
- 三段式
- 过度排比
- AI 腔
- 节奏太平
- 假装完整
- 过度解释
- 宣传腔
- 没有人味

输出：

- 原始正文
- 评分
- 命中问题
- 问题片段
- 修改方向
- 优化版

禁止：

- 不允许只说“已优化”
- 不允许只替换词
- 不允许优化后跑题
- 不允许删除源头帖和评论问题的绑定关系

### 7. 确认后生产

只有用户在网页端确认最终文案后，才解锁后续生产。

可解锁：

- 小红书卡片图
- 配图提示词
- 正文插图
- 公众号封面
- 视频脚本
- 小妹视频工作台任务
- HTML/PNG 卡片

仍需展示：

- 使用哪条源头帖
- 使用哪个标题
- 使用哪个正文版本
- 图片或视频依据
- 输出路径
- 测试证据
- 截图或成品链接

禁止：

- 未确认文案前生成图片
- 未确认文案前生成视频
- 未确认文案前打包
- 未确认文案前上线
- 未确认文案前改服务器或生产配置

## 验收方式

每一步都必须有独立验收。

验收格式：

```text
当前步骤：
输入来源：
选择依据：
页面输出：
禁止项是否锁定：
测试命令：
截图或文件路径：
下一步是否等待确认：
```

没有通过当前步骤，不进入下一步。
## 目标定义门禁

每一步开始前必须先写清楚三件事，缺一项就不继续：

1. 目标：这一轮到底要产出什么，不能用“优化一下”“继续做”这种模糊目标。
2. 停止条件：什么时候必须停下来等人确认，什么时候算失败，什么时候不能继续烧 token。
3. 验收证据：页面上能看到什么，接口返回什么，测试命令是什么，截图或文件路径在哪里。

这条规则来自当前项目复盘：AI 不是越自动越好，真正有价值的是把目标、边界和验收标准定义清楚。Longka 内容工厂每一步都必须把“输入、选择依据、草稿、确认、执行、证据”产品化，避免模型自由发挥、固定模板兜底和无意义重试。
