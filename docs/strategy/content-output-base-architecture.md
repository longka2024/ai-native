# AI Native 内容输出基座融合架构

更新时间：2026-05-28

## 一句话定位

我们的 AI Native 系统不是堆工具，而是把“文字内容输出”和“视频内容输出”做成两条稳定生产线：输入业务目标和素材，经过 SOP、模型、工具、质检、反馈，最后产出能交付、能发布、能复盘的内容资产。

## 总体原则

1. 人做判断，AI 做执行和调度。
2. 每个 Agent 都必须有明确 SOP：能做什么、不能做什么、输入是什么、输出是什么、验收标准是什么。
3. 工具不是直接暴露给客户，而是包装成老板看得懂的“AI 员工能力”。
4. 每次产出都要留下结构化记录，变成下一次优化的企业上下文。

## 文字内容输出层

目标：把老板的模糊想法变成可发布、可成交、可复用的文字资产。

核心链路：

1. 信息源采集：热点、竞品、评论区、私域反馈、客户问答、行业文章。
2. 素材清洗：去重、分类、提炼痛点、标注人群和场景。
3. 选题判断：判断内容是认知型、种草型、转化型、案例型还是复盘型。
4. 文案生成：根据平台、账号定位、目标用户、发布目的生成多版本文案。
5. 格式适配：小红书、朋友圈、公众号、短视频口播、销售话术、报告页。
6. 质量检查：是否空泛、是否跑题、是否有真实案例、是否有行动引导。
7. 入库复盘：记录哪种选题、标题、结构、配图带来更好反馈。

已纳入的技术基座：

- Waza：方法论沉淀、自我迭代、去冗余。
- Kami：把内容包装成报告、卡片、页面、交付物。
- md2wechat-skill：Markdown 到公众号排版。
- write / humanizer-zh：中文润色、去 AI 味。
- read / learn / research 类能力：资料阅读、文章拆解、信息归纳。
- GEOFlow：面向 GEO/搜索可见性的内容结构参考。
- MediCrawler / 平台采集类工具：作为信息源采集层参考。

文字层交付形态：

- 朋友圈配文
- 小红书笔记
- 短视频口播稿
- 公众号文章
- 客户成交话术
- 行业观察报告
- 小程序内的分享文案

## 视频内容输出层

目标：把文字、图片、报告、案例素材变成可发布的短视频资产。

核心链路：

1. 明确视频类型：单客户报告、多案例宣传、产品流程讲解、发圈种草、老板业务案例。
2. 素材分层：原始上传图、试看图、完整 11 张报告图、案例库图、产品流程图、封面图。
3. 文案匹配：每个模板必须匹配对应素材，不允许文不对题。
4. 视频模板生成：Remotion 负责稳定主视频结构。
5. 高级动效增强：HyperFrames 负责片头、标题卡、转场、CTA。
6. 音频生成：MiniMax 旁白、KIE/Suno 背景音乐。
7. 合成导出：FFmpeg 拼接、混音、转码、抽封面。
8. 质检重剪：video-use 做关键帧检查、字幕检查、素材匹配检查、重剪建议。
9. 批量变体：吸收 MoneyPrinterTurbo 的批量生成、字幕预设、节奏控制和素材门禁。

已纳入的技术基座：

- Remotion：主视频模板和可参数化成片。
- HyperFrames：高级片头、标题卡、转场、CTA 动效。
- FFmpeg：拼接、压制、混音、封面抽帧。
- video-use：质检、重剪、时间线分析。
- MoneyPrinterTurbo：批量视频生产工程参考。
- KIE/Suno：背景音乐。
- MiniMax TTS：旁白。

视频层交付形态：

- 小红书种草视频
- 朋友圈发圈视频
- 小程序功能演示视频
- 单客户报告展示视频
- 多案例宣传视频
- U 盘交付里的本地视频工作台

## 客户可见的产品表达

客户不应该看到一堆技术名词，而应该看到：

- 文案员工：帮你写朋友圈、小红书、公众号、短视频口播。
- 视频员工：帮你做封面、旁白、配乐、成片。
- 选题员工：每天帮你找值得发的选题。
- 资料员工：帮你整理客户反馈、评论区、私域线索。
- 运营员工：帮你排发布计划、复盘数据、提出下一步动作。

底层是技术基座，前台是 AI 员工。

1. 色彩项目的小妹视频工作台：先把视频生产线跑稳定，做到能选素材、选模板、生成文案、生成旁白、生成音乐、合成视频、质检成片。
2. AI Native 内容中台：先把文字输出链路做扎实，做到从信息源到选题、文案、平台格式、发布清单都有可见页面。

这两件事打通后，才能自然扩展到老板的 AI 员工系统、U 盘交付、小程序版和网站端。
