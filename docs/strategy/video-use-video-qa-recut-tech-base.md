# video-use 视频质检与重剪技术基座

更新时间：2026-05-28

## 定位

`video-use` 不作为小妹视频工作台的主模板引擎。主模板仍由 Remotion 承担，片头、转场、标题卡由 HyperFrames 承担，最终拼接和混音由 FFmpeg 承担。

`video-use` 的正确位置是：成片质检、时间线分析、重剪、二次优化层。

## 解决的问题

当前视频工作台最大的问题不是“能不能生成视频”，而是：

- 第一眼没有点击欲望。
- 画面素材和文案不匹配。
- 不同客户素材混用，导致观感不可信。
- 字幕、标题、图片之间可能遮挡。
- 成片失败后没有清晰原因。
- 小妹无法判断视频哪里不好，只能反复试。

`video-use` 应用于这些问题的后处理：

- 抽取关键帧，检查前 3 秒是否有明确钩子。
- 检查字幕是否压住人脸、报告页重点区域。
- 根据时间线识别哪些素材被使用、出现多久。
- 对失败视频做重剪，例如删掉弱画面、缩短空镜、重排节奏。
- 输出可读的 QA 报告，告诉操作者“哪里需要换素材、换文案、换模板”。

## 接入小妹工作台的方式

小妹工作台后续应增加一个“视频质检”按钮，生成视频后自动跑一次检查。

质检结果显示为：

- 首 3 秒评分：钩子是否明确、封面是否可点击。
- 素材匹配：当前视频是单客户报告、多案例宣传，还是混合宣传。
- 画面风险：是否有文字遮挡、图片过小、画面空、同质化严重。
- 音频风险：旁白是否存在、背景音乐音量是否过低或过高。
- 修改建议：换哪组素材、换哪个模板、是否需要重新生成封面。

## 与现有视频基座的关系

- Remotion：负责稳定、可参数化的主视频模板。
- HyperFrames：负责高级片头、标题卡、转场、CTA 动效。
- FFmpeg：负责拼接、混音、转码、封面抽帧。
- KIE/MiniMax：负责配乐、旁白。
- MoneyPrinterTurbo：提供批量变体、字幕、节奏、素材门禁的工程参考。
- video-use：负责成片后的分析、重剪、二次优化和质量反馈。

## 强约束

1. 单客户报告视频不能混入其他人的照片，除非明确标注为“案例参考”。
2. 预览素材只有两张原图加一张试看图时，不能套用完整报告拆解模板。
3. 完整报告素材必须能识别 2 张原始图和 11 张报告图，模板才能做发型、妆容、穿搭、色彩分镜。
4. 字幕和贴纸必须最后叠加，避免被后续转场破坏。
5. 所有输出必须生成一份素材使用清单，便于复盘。

## 下一步

把 `video-use` 的能力包装成小妹工作台的“质检与重剪”模块：

1. 生成视频后自动抽 1 秒、3 秒、5 秒、中段、结尾帧。
2. 生成 `video-qa.json`，记录素材、模板、音频、字幕、时长。
3. 页面显示“通过 / 需要修改 / 不建议发布”。
4. 对“不建议发布”的视频提供一键重剪入口。
