# Longka AI Native 团队操作手册

## 1. 手册目的

这份手册给团队成员使用，解决三个问题：

- 今天打开系统后该做什么
- 每一步做到什么标准才算完成
- 采集、入库、创作、出图、复盘之间怎么衔接

Longka AI Native 不是让用户面对一堆功能按钮自己摸索，而是要把每天的内容工作变成可执行的 SOP。

## 2. 每日工作台原则

每日工作台是主入口。

用户不应该在多个页面之间来回猜。核心流程必须在工作台里完成：

```text
选择目标
-> 选择素材来源
-> 采集或读取资产
-> 筛选可用素材
-> 生成母题
-> 选择标题
-> 生成文案
-> 优化确认
-> 生成图文/视频
```

其他页面只能作为资产管理、历史记录和配置中心，不能打断当天工作流。

## 3. 每天怎么用

### 第一步：确定今天的发布目标

必须先明确：

- 行业：例如 AI 自媒体、大健康、美业护肤。
- 平台：小红书、公众号、视频号、今日头条、X。
- 内容目标：涨粉、引流、成交、教育用户、建立专业感。
- 关键词：今天围绕什么方向找素材。

没有目标，后面的采集和生成都会发散。

### 第二步：选择素材来源

素材来源分三类：

1. 新采集
   从指定平台、关键词、账号或链接获取最新内容。

2. 历史资产
   从已经入库的内容资产中筛选可复用素材。

3. 手工导入
   用户粘贴文章、链接、截图、X 书签、Obsidian 笔记、飞书表格内容。

系统必须明确告诉用户当前用的是哪一种来源，不能把历史数据伪装成新采集数据。

## 4. 采集模块 SOP

采集模块不是临时补流程的工具，而是全局内容资产入口。

每次采集必须记录：

- 采集平台
- 采集方式
- 关键词或账号
- 采集时间
- 采集批次
- 成功数量
- 失败数量
- 过滤数量
- 入库数量
- 原链接
- 作者
- 发布时间
- 标题 / 正文
- 互动数据
- 评论数据

页面必须显示采集动态：

- 正在打开页面
- 正在搜索关键词
- 正在读取帖子列表
- 正在筛选高质量帖子
- 正在抓取详情
- 正在抓取评论
- 正在写入内容资产库
- 本次完成数量

不能只显示 0 或空白等待。

## 5. 内容资产库 SOP

所有进入内容资产库的数据，都必须是真实可追溯数据。

每条资产至少包含：

- 来源平台
- 来源链接
- 作者
- 原文标题
- 原文正文
- 发布时间
- 互动数据
- 内容分类
- 适合平台
- 可复用角度
- 是否入母题库
- 是否已用于创作

内容资产库不是“采集结果列表”，而是后续选题、标题、文案、训练模型的基础。

## 6. 母题筛选 SOP

母题不是随便把原帖改写。

一条素材能成为母题，至少要满足其中几项：

- 有明确用户问题
- 有强观点
- 有反常识
- 有可讨论冲突
- 有真实经验
- 有可收藏的信息结构
- 有评论区需求
- 有平台传播潜力

如果素材质量一般，系统要提示“只适合作为参考，不建议直接二创”。

## 7. 标题生成 SOP

标题不能固定套几种句式。

标题库要按类型沉淀：

- 痛点型
- 反常识型
- 经验复盘型
- 自查型
- 对比型
- 避坑型
- 清单型
- 趋势判断型
- 结果承诺型
- 评论区问题型

生成标题时，系统要先判断母题适合哪几类标题，再从标题资产库和标题公式库里匹配。

标题候选必须有差异：

- 角度不同
- 人群不同
- 情绪不同
- 承诺不同
- 内容结构不同

不能只是把同一套标题换几个词。

## 8. 文案生成 SOP

文案生成必须遵循：

```text
先判断用户为什么关心
-> 再确定主问题
-> 再选择内容结构
-> 再写初稿
-> 再做 AI 味检查
-> 再做平台适配优化
```

不能直接套模板。

一篇合格文案至少要做到：

- 标题和正文主问题一致
- 开头有真实语境
- 正文有信息增量
- 结构服务于主题
- 语言口语但不低级
- 有用户视角
- 有明确下一步动作
- 不乱编身份、数据、案例

## 9. 优化 SOP

“继续优化一次”不能只在文末加建议。

每次优化必须真实改写正文，至少改变以下几项之一：

- 开头进入方式
- 主张句
- 段落顺序
- 例子和细节
- 表达口语化
- 结尾转化动作
- 标题与正文贴合度

优化后要告诉用户：

- 改了哪里
- 为什么改
- 比上一版好在哪里

## 10. 图文 / 视频 SOP

文案未确认前，不进入图文和视频生产。

确认文案后，再判断适合：

- 小红书图文卡片
- 小红书封面
- 公众号长图
- 视频号口播脚本
- 短视频分镜
- 直播间话术

图文不是把正文贴到图片上，而是把内容拆成可阅读、可停留、可收藏的视觉结构。

## 11. 复盘 SOP

发布后要记录：

- 发布时间
- 平台
- 标题
- 封面
- 阅读量
- 点赞
- 收藏
- 评论
- 转发
- 私信
- 转化
- 是否继续复用

复盘数据必须回流到内容资产库，形成正例、反例和平台指纹。

## 12. 团队验收标准

一个流程算跑通，必须同时满足：

- 用户知道下一步点哪里
- 页面显示当前正在做什么
- 结果来自真实数据
- 资产能沉淀
- 创作能复用资产
- 文案质量能优化
- 结果能继续进入图文/视频
- 发布后能回流

只要用户看不懂下一步，就不算完成。
