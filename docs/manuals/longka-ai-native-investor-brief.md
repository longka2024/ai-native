# Longka AI Native 投资人简报

## 1. 项目定位

Longka AI Native 不是一个提示词工具，也不是单点 AI 写作产品。

它的定位是：面向垂直行业的 AI Native 工作流产品工厂。

第一阶段从内容生产切入，因为内容生产同时具备三个特点：

- 客户痛点高频：每天都要找选题、写内容、做图文、做视频、复盘数据。
- AI 替代感强但实际效果差：客户已经用过豆包、DeepSeek、ChatGPT，但普遍觉得 AI 味重、没流量、不可持续。
- 数据闭环清晰：采集爆款样本、沉淀内容资产、生成内容、发布复盘，可以形成持续进化的系统。

Longka AI Native 的长期目标，是把一个行业里反复发生的工作流程，拆成可采集、可判断、可执行、可复盘、可训练的 AI Native 系统。

## 2. 立项脉络

2026-05-15，项目开始从“做一个 AI 工具”转向“用 AI Native 方法做行业工作流产品”。

2026-05-16，Longka 被明确为“AI Native 行业工作流产品工厂”：不是为单个需求写一次代码，而是把行业里的可重复流程产品化。

早期阶段主要积累技术基座：

- Web 工作台
- 服务器部署环境
- PostgreSQL 数据库
- MediaCrawlerPro 采集能力
- XCrawl 采集能力
- 图文生成能力
- 视频生成能力
- Codex / skills / harness / superpowers 工作流
- 本地包与团队测试环境

这个阶段的教训也很明确：只有工具不够。没有系统级使用手册、产品流程标准和开发约束，AI 会很容易把项目做偏，变成不断打补丁。

因此，项目第二阶段的核心不是继续堆工具，而是建立 Longka 自己的操作系统。

## 3. 核心产品逻辑

Longka AI Native 内容系统的主流程是：

```text
行业 + 目标 + 平台 + 关键词
-> 真实采集 / 人工导入
-> 内容资产库
-> 选题候选
-> 选择母题
-> 标题候选
-> 选择标题
-> 基于素材生成初稿
-> 质量诊断与改写进化
-> 用户确认
-> 图文 / 视频生产
-> 发布复盘
-> 数据回流资产库
```

这条流程的关键不是“让 AI 写一篇文章”，而是构建一个可以持续学习的内容生产系统。

## 4. 产品模块

Longka AI Native 当前按六个模块建设：

1. 采集模块
   负责从小红书、X、公众号、头条、RSS、Newsletter、网页等来源获取真实素材。

2. 内容资产模块
   把采集结果清洗、去重、筛选、标注，沉淀成可复用的资产，而不是临时页面数据。

3. 内容制作模块
   基于内容资产做选题、标题、文案、脚本、改写和质量评分。

4. 图文 / 视频生产模块
   在文案确认后生成小红书图文卡片、封面、视频脚本和视频素材。

5. 发布复盘模块
   记录发布平台、发布时间、阅读、互动、转化结果，并回流到内容资产库。

6. 客户项目模块
   不同客户、行业、平台、账号、素材和结果隔离管理，形成长期可运营系统。

## 5. 数据壁垒

Longka 的壁垒不在“调用哪个大模型”，而在数据资产和工作流闭环。

未来核心资产包括：

- 平台高表现正例库
- 平台低表现反例库
- 2019 年前人工语料库
- 标题公式库
- 开头公式库
- 爆点结构库
- 用户问题库
- 评论区痛点库
- 行业案例库
- 发布反馈库
- 平台指纹库
- 人味表达特征库

这些资产会支撑后续本地小模型训练和内容评分系统。

大模型负责生成，小模型和规则系统负责诊断、评分、筛选、改写建议和平台指纹约束。

## 6. 为什么不是普通 AI 写作工具

普通 AI 写作工具的问题是：

- 没有真实素材来源
- 没有平台样本库
- 没有正反例对比
- 没有用户问题沉淀
- 没有发布反馈回流
- 没有质量评分闭环
- 没有团队 SOP

Longka 要解决的是完整生产链路：

```text
找什么写
为什么值得写
模仿谁的成功结构
如何写得像人
如何适配平台
如何做成图文/视频
发出去效果如何
下次怎么更好
```

## 7. 阶段目标

第一阶段目标：跑通内容生产闭环。

- 真实采集
- 内容入库
- 选题生成
- 标题生成
- 文案生成
- 质量优化
- 图文生产
- 团队可测试

第二阶段目标：建立内容资产库和标准 SOP。

- 对标账号库
- 标题库
- 爆文拆解库
- 用户问题库
- 样本标注库
- 每日工作台
- 采集任务管理

第三阶段目标：建立平台指纹和训练系统。

- 正例 / 反例样本库
- 人工语料库
- 内容标签体系
- Qwen 3B 本地模型评分
- 平台爆款指纹
- AI 味 / 人味检测
- 发布数据回流

## 8. 当前判断

Longka AI Native 的方向是成立的。

但它不能再靠“想到哪改到哪”的方式推进。项目必须进入文档化、标准化、验收化阶段。

从现在开始，所有关键能力都必须满足：

- 有产品流程
- 有数据结构
- 有页面动作
- 有用户引导
- 有真实验证
- 有复盘记录
- 有可复用手册

这套手册体系，就是 Longka 从工具堆叠进入产品化阶段的开始。
