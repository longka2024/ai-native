# Longka 内容资产系统手册

## 1. 为什么需要内容资产系统

Longka AI Native 不能只做“输入关键词 -> AI 写一篇”。

真正有长期价值的是内容资产系统：

```text
持续采集好内容
-> 拆解成可复用资产
-> 支撑选题、标题、文案、图文、视频
-> 发布后回收效果数据
-> 形成平台指纹和训练语料
```

没有内容资产库，AI 写作就只能靠模型的通用知识和提示词，结果很容易空、顺、假、模板化。

有内容资产库，系统才知道：

- 最近行业里什么话题有热度
- 用户评论区真正问什么
- 哪类标题更容易被点开
- 哪种结构更容易收藏
- 哪些表达更像人
- 哪些内容发出去有效
- 哪些内容只是看起来完整但没传播力

## 2. 内容资产系统的定位

内容资产系统不是资料夹，也不是简单的采集结果列表。

它是 Longka 的内容生产底座。

它连接四件事：

1. 采集
   从小红书、X、公众号、头条、RSS、Newsletter、网页、Obsidian、书签等来源获取素材。

2. 拆解
   把原始内容拆成标题、开头、结构、爆点、用户痛点、案例、观点、表达方式。

3. 复用
   在创作时调用这些资产，生成新的选题、标题、文案、脚本、图文卡片。

4. 训练
   积累正例、反例、人工语料和发布反馈，用于后续本地模型训练和平台指纹分析。

## 3. 资产类型

内容资产库至少包含以下几类资产。

### 3.1 原始样本库

保存采集或导入的原文。

字段包括：

- 平台
- 作者
- 原链接
- 标题
- 正文
- 图片说明
- 发布时间
- 抓取时间
- 采集批次
- 采集方式
- 原始互动数据
- 原始评论数据

原则：原文必须可追溯，不能丢来源。

### 3.2 用户问题库

从评论区、私信、问答、客户反馈、社群聊天中提取用户真实问题。

字段包括：

- 问题原文
- 来源平台
- 关联帖子
- 用户场景
- 痛点类型
- 情绪强度
- 是否适合做选题
- 是否适合做转化入口

用户问题库的价值，是让内容从“我想说什么”变成“用户正在问什么”。

### 3.3 标题资产库

保存真实平台高表现标题和人工整理的标题公式。

标题不能一股脑存，要分类：

- 痛点型
- 反常识型
- 经验复盘型
- 自查型
- 对比型
- 避坑型
- 清单型
- 趋势判断型
- 结果承诺型
- 评论区问题型
- 人群点名型
- 失败复盘型

每个标题要记录：

- 标题原文
- 标题类型
- 适合平台
- 适合行业
- 触发情绪
- 承诺方式
- 是否可套用
- 套用风险
- 来源链接
- 表现数据

标题库的目标不是机械套公式，而是在创作时匹配最适合当前母题的标题角度。

### 3.4 爆文拆解库

爆文拆解库保存一篇内容为什么值得学。

拆解维度包括：

- 大分类
- 小分类
- 用户为什么要看
- 看完能得到什么
- 首屏留人机制
- 标题钩子
- 开头方式
- 正文结构
- 信息增量
- 情绪起伏
- 案例使用
- 转化动作
- 可复用公式
- 不能照搬的部分

拆解库的目标不是复刻原文，而是提炼可复用机制。

### 3.5 母题库

母题是可以被多平台、多形式复用的核心内容主题。

一条母题可以衍生：

- 小红书图文
- 公众号长文
- 今日头条文章
- 视频号脚本
- 朋友圈文案
- 直播间话术
- 私域转化话术

母题字段包括：

- 母题标题
- 核心问题
- 目标人群
- 使用场景
- 来源证据
- 可用素材
- 适合平台
- 可扩展形式
- 风险边界

母题不是原帖标题，也不是一句空话。它必须能支撑后续创作。

### 3.6 表达资产库

保存更像人的表达方式。

包括：

- 口语化句式
- 真实经验表达
- 转折方式
- 犹豫感
- 不那么工整的句子
- 人类写作常见断裂
- 行业真实说法
- 客户常用词

表达资产库用于降低 AI 味。

### 3.7 正反例样本库

正例：高阅读、高互动、高收藏、高转化内容。

反例：低阅读、低互动、看起来完整但没效果的内容。

正反例必须分平台、分行业、分内容类型保存。

字段包括：

- 平台
- 行业
- 内容类型
- 阅读量
- 点赞
- 收藏
- 评论
- 转发
- 转化
- 爆款标签
- 低效原因
- 人工判断

正反例库服务于平台指纹分析和本地小模型训练。

### 3.8 发布复盘库

保存 Longka 自己生成并发布过的内容结果。

字段包括：

- 客户
- 平台
- 标题
- 正文版本
- 图片/视频版本
- 发布时间
- 阅读量
- 点赞
- 收藏
- 评论
- 转化
- 复盘结论
- 是否进入正例库
- 是否进入反例库

发布复盘库是系统持续进化的关键。

## 4. 入库标准

不是所有采集内容都应该进入核心资产库。

入库分三层：

1. 原始暂存
   只要真实采集成功，就先保存原始数据。

2. 可用资产
   经过过滤后，确认有参考价值，进入内容资产库。

3. 精品资产
   经过人工或模型判断，确认有高复用价值，进入标题库、拆解库、母题库或正例库。

过滤标准包括：

- 是否来自目标平台
- 是否来自目标账号或关键词
- 是否有原链接
- 是否内容完整
- 是否和行业相关
- 是否互动数据可信
- 是否具备用户问题
- 是否有观点或结构
- 是否适合复用

## 5. 采集到入库的流程

标准流程：

```text
创建采集任务
-> 获取原始内容
-> 保存原始样本
-> 去重
-> 清洗
-> 质量评分
-> 分类
-> 提取标题/问题/结构/案例
-> 入内容资产库
-> 可选入母题库
-> 可选生成拆解卡
```

每一步都要有状态，不允许用户不知道系统在干什么。

## 6. 从资产到创作的流程

标准流程：

```text
选择平台和目标
-> 从资产库筛选候选素材
-> 生成母题候选
-> 用户选择母题
-> 匹配标题资产
-> 生成差异化标题
-> 用户选择标题
-> 生成正文
-> 内容体检
-> 改写进化
-> 用户确认
-> 进入图文/视频生产
```

创作时必须绑定来源资产，不能脱离素材胡写。

## 7. 多平台一鱼多吃

一条好母题不应该只服务一个平台。

系统要支持“一鱼多吃”：

- 小红书：短、强钩子、图文结构、收藏价值。
- 公众号：长、逻辑完整、故事和观点展开。
- 今日头条：信息密度、争议点、通俗解释。
- 视频号：口播节奏、强开头、情绪推进。
- 朋友圈：短、信任感、成交语境。

同一母题可以生成不同平台版本，但不能简单复制粘贴。

## 8. 与本地模型训练的关系

内容资产库未来要服务本地 Qwen 3B 等小模型。

训练不是第一天就做。

正确顺序是：

```text
真实样本足够多
-> 字段结构稳定
-> 标签体系稳定
-> 正反例明确
-> 人工语料充足
-> 再做小模型训练
```

小模型优先承担：

- 分类
- 打分
- 相似度判断
- AI 味识别
- 人味评分
- 平台指纹匹配
- 改写建议

最终生成仍可以由大模型完成。

## 9. UI 呈现原则

内容资产库页面不能密密麻麻堆字段。

用户最关心的是：

- 这条素材从哪里来
- 为什么值得用
- 适合做什么
- 下一步能点什么
- 有没有风险

每张资产卡应优先展示：

- 来源
- 作者
- 原标题
- 核心观点
- 用户问题
- 推荐用途
- 入库状态
- 下一步按钮

内部字段可以折叠，不要压迫用户。

## 10. 验收标准

内容资产系统第一阶段验收标准：

- 能真实采集或导入素材
- 能保存原始样本
- 能区分新采集和历史资产
- 能筛选出可用母题
- 能生成标题候选
- 能生成绑定素材的正文
- 能保存拆解结果
- 能回到今日工作台继续创作
- 用户知道下一步该做什么

第二阶段验收标准：

- 支持多平台资产
- 支持对标账号库
- 支持标题库分类
- 支持用户问题库
- 支持发布复盘
- 支持正反例标注
- 支持平台指纹评分

第三阶段验收标准：

- 支持大样本训练语料导出
- 支持本地小模型评分
- 支持 AI 味 / 人味检测
- 支持发布效果回流
- 支持客户项目隔离

## 11. Longka 的底线

内容资产系统必须守住几条底线：

- 不用假数据冒充真实采集。
- 不把历史资产说成新采集。
- 不采集后只展示，必须能进入创作。
- 不让用户面对一堆看不懂的字段。
- 不把资产库做成死资料库。
- 不让文案脱离资产胡写。
- 不把平台差异抹平。
- 不在样本质量不足时盲目训练模型。

Longka 的内容资产库，最终要成为内容生产、客户交付和模型训练的共同基础。
