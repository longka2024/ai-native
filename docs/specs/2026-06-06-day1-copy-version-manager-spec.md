# Day1 文案版本管理器 Spec

## 1. 背景

当前内容制作模块最大的问题不是采集，也不是图文闭环，而是文案出来后不能稳定迭代：

- 继续优化容易只追加建议。
- 用户切换或优化后，上一版稿子容易丢。
- 用户看不到每一版有什么变化。
- 用户不能明确恢复或确认某一版。
- 文案未确认时，后续图文/视频不应该解锁。

Day1 只解决“文案版本管理 + AI 继续优化”。

## 2. 目标

交付一个可用的文案版本管理器。

用户生成正文后，可以：

1. 自动保存第 1 版。
2. 点击继续优化，让 AI 基于当前正文真实改写出下一版。
3. 查看每一版。
4. 恢复任意一版。
5. 确认任意一版。
6. 确认后不丢稿，并解锁后续制作入口。

## 3. 必须做到

### 3.1 文案版本历史

每次生成或优化出完整正文后，系统必须保存一个版本。

每个版本至少包含：

- versionId
- round
- title
- copy
- score
- label
- createdAt
- 是否当前查看
- 是否已确认

### 3.2 继续优化接 AI 改写

点击继续优化时，必须把以下信息传给后端：

- 当前标题
- 当前完整正文
- 当前版本 id
- 当前版本分数
- 优化方向
- Longka 文案体检反馈
- 源头素材信息

AI 必须基于当前正文改写，不允许只在文末追加建议。

### 3.3 每一版可回看

版本列表里必须能点开某一版，查看该版完整标题和正文。

查看旧版时，不应删除其他版本。

### 3.4 每一版可恢复

用户点击“恢复此版”后：

- 当前标题变成该版本标题。
- 当前正文变成该版本正文。
- 文案体检切换到该版本。
- 当前版本标记更新。
- 后续优化基于恢复后的版本继续。

### 3.5 每一版可确认

用户点击“确认此版”后：

- 该版本标记为已确认。
- 当前正文固定为该版本。
- 后续图文/视频入口解锁。
- 如果用户之后再次优化，必须取消确认状态，避免旧确认误用。

## 4. 禁止项

Day1 禁止扩大范围。

不做：

- 采集模块改造
- 母题筛选改造
- 标题库重构
- 内容资产模块统一
- 图文卡片生成
- 视频生成
- 今日工作台整体重构
- 122 部署，除非用户明确说部署验证

## 5. 验收标准

本地验收至少验证：

1. 生成正文后出现第 1 版。
2. 点击继续优化后新增第 2 版。
3. 第 1 版仍可回看。
4. 第 1 版可恢复。
5. 第 2 版可确认。
6. 确认后页面显示已确认状态。
7. 再次优化会取消旧确认。
8. 优化结果不是只在文末追加建议。

## 6. 后续唤醒方式

后续任何时候，用户说以下话，都必须先读本文件：

- “按 Day1”
- “按文案版本历史”
- “继续优化不要只追加建议”
- “不丢稿”
- “可回看、恢复、确认”
- “你又跑偏了”

读完本文件后，只允许按本文件范围开发。
