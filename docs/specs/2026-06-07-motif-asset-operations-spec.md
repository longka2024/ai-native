# 母题资产运营与一鱼多吃平台复用 Spec

## 1. 名字

母题资产运营与一鱼多吃平台复用 Spec

## 2. 触发词

后续用户或开发讨论里出现以下说法时，必须先读取本 spec：

- “母题资产运营”
- “一鱼多吃”
- “小妹工作流”
- “按母题资产 spec”
- “平台切换不跑题”
- “同一个话题做小红书、公众号、视频号”
- “内容资产沉淀后怎么复用”

## 3. 用户行为目标

本 spec 解决的是一个真实运营人员的日常动作链：

1. 运营人员监控对标账号、历史收藏、RSS、网页、公众号、X、小红书等信息源。
2. 看到若干值得写的帖子或文章。
3. 选中其中一个话题作为“母题”。
4. 自己决定先做哪个平台，不强制小红书优先。
5. 针对目标平台生成合适内容：小红书图文笔记、公众号长文、视频号 / 抖音脚本、朋友圈文案。
6. 做完一个平台后，可以继续用同一个母题切换到其他平台。
7. 切换平台时必须保持同一个母题，不跑题，不照搬旧平台结构，不产生明显 AI 味。
8. 配图必须跟随平台和内容语义处理：小红书做图集，公众号按语义插入文章，视频转为分镜参考，朋友圈轻量复用或重生成。
9. 发布后允许录入或采集复盘数据，反哺母题资产、标题库、结构库、案例库和后续改写。

## 4. 范围

本 spec 允许做以下事情：

- 把“内容资产”从单个平台成品，升级为平台中立的“母题资产”。
- 每个母题资产保留来源材料、选题理由、用户痛点、核心主张、可复用角度。
- 今日工作台中允许用户选择首发平台。
- 完成一个平台内容后，保存为该母题下的一个平台版本。
- 内容资产卡片必须展示下一步动作，而不是只展示数据：继续做小红书、改成公众号长文、改成视频脚本、改成朋友圈文案、查看复盘数据。
- 平台切换必须进入对应平台的标题、正文、脚本或图文生产流程。
- 小红书图文确认后，允许生成真实图片，不允许用 CSS 假图冒充最终图。
- 公众号长文预览必须能把已有配图按语义插入文章段落。
- 视频脚本预览必须至少包含口播结构、分镜方向和封面/配图建议。
- 复盘卡必须说明数据从哪里来：手动录入、平台采集、暂未采集、demo/样例必须明确标记。
- 刷新页面后，当前浏览器内已经保存的成品和母题状态不能无故丢失。

## 5. 禁止范围

本 spec 明确禁止以下做法：

- 不允许默认“小红书永远是第一步”。
- 不允许把小红书文案直接复制成公众号长文或视频脚本。
- 不允许平台切换后跑题，只借用标题或几句空泛观点。
- 不允许把内部过程语句、分析标签、操作提示写进面向用户的最终图片或文案。
- 不允许空按钮、点了没反应的按钮、跳转后用户不知道下一步做什么。
- 不允许新开一个割裂页面作为唯一入口，除非用户单独确认。
- 不允许把采集工程、资产库、创作流程做成互相孤立的工具。
- 不允许把本地浏览器 localStorage 说成团队共享资产库。
- 不允许把 CSS 卡片、占位图、假链接当成最终可发布配图。
- 不允许未确认最终文案就开放最终批量出图、视频渲染、打包、发布。
- 不允许在用户要求 122 验收时只用本机 127.0.0.1 证明完成。

## 6. 验收标准

本 spec 对应功能必须满足以下验收条件：

1. 用户可以从一个母题资产选择任意首发平台，不被强制从小红书开始。
2. 小红书、公众号、视频脚本、朋友圈至少有明确的入口和下一步动作说明。
3. 保存后的成品出现在内容资产中，并能看出它属于哪个母题、哪个平台版本。
4. 同一个母题可以继续生成另一个平台版本。
5. 切换平台后，标题、正文、结构和图片策略随平台变化，而不是只换标题。
6. 小红书图文必须显示真实生成图片或明确失败原因。
7. 公众号长文必须能展示图片插入位置，而不是跳回小红书编辑态。
8. 视频脚本必须显示脚本/分镜预览，而不是跳回小红书编辑态。
9. 内容资产卡片必须告诉用户“现在能做什么”，包括复用、复盘、继续生产。
10. 复盘数据区必须说明数据来源；没有真实数据时不能伪装成已采集。
11. 刷新页面后，浏览器内已保存资产不应无故消失。
12. 122 验收必须通过公开地址验证，不以本机页面为准。

## 7. 当前已知技术边界

- 122 Web 版本后续应以 PostgreSQL 作为团队共享内容资产和训练语料的正式来源。
- 当前浏览器本地保存只能作为阶段性工作台缓存，不能作为最终团队资产库。
- 43 是正式图像生成服务方向；本机可做源码开发和临时验证，但最终图文资产不能落在 C 盘。
- 采集模块、内容资产模块、内容制作模块、图文/视频生产模块、发布复盘模块是后续产品大模块，但操作体验必须在一个连续工作台里完成。

## 8. 开发前复述要求

每次开发本 spec 相关功能前，AI 必须复述：

- 本次只改哪一个用户动作链。
- 本次不改哪些相关模块。
- 本次验收在哪个环境完成：local-source-dev、122-web、43-generation。
- 是否会生成资产、部署、重启服务、触碰生产配置。
- 如果需要扩展范围，先停下来重新对齐 spec。
