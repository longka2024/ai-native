# SPEC — 口播视频生产线（数字人 + 首尾帧动画主体 + 字幕）

> 状态：北极星已定，分期实现
> 日期：2026-06-18
> 关联：`2026-06-18-video-keyframe-line-spec.md`、`2026-06-17-video-line-spec.md`

## 北极星（终极形态，用户定）

一条口播脚本 → 自动产出：
- **右下角：用户本人数字人口播**（用户形象在讲这件事，对口型，可用本人声音克隆）；
- **主体画面：紧扣主题、真正动起来的视频**（首尾帧补间 AI 动画，不是静图/PPT）；
- **+ 动态字幕**；
- 合成为一条竖屏成片，长度 = 口播时长。

> "视频视频，得动起来" + "右下角我出镜数字人，主体画面扣主题动起来，这才是最好的"。

## 根源矛盾（必须正视）

口播视频要同时满足：① 画面真动 ② 盖住整段口播 ③ 便宜 —— AI 视频按时长/段收费，三者天然打架。
**解法不是假动（Ken Burns/翻页 PPT，用户已否决——"意义不大"），而是：**
- **镜头少**（归并成 ~5 个关键画面，不要一句一镜）；
- **口播精简**（脚本控制在 ~30s，5 个 beat：钩子/问题/答案/对比/收尾）；
- **真动靠首尾帧补间**（给首图+尾图，AI 补中间运动，可控）。

## 关键机制：首尾帧补间（用户指定的"动"法）

每个分镜：**首帧图 + 尾帧图 → 图生视频模型补出中间运动**。
- 优点：画面真动、运动可控、对版稳。
- 模型要求：必须支持 `first_frame + last_frame`。
  - 已知支持：Seedance 2 / fast（**按秒**计费，较贵）。
  - **待你后台确认**：是否有"支持首尾帧 + 按次计费(便宜)"的模型（Kling/Vidu/Hailuo 首尾帧变体等）。我网页查到的价是过时的（Sora2 都下架了），**你后台看到的才算数**。
- 兜底：找不到便宜的首尾帧模型，就用单图→视频（Hailuo，便宜，AI 自动运动，也真动，但只控开头）。

## 数字人口播（北极星组件，phase 后置）

- 用户形象 → 数字人 avatar，对口型到口播音轨 → 谈话头视频 → 合成到主体画面右下角（ffmpeg 画中画）。
- 声音：本人声音克隆（MiniMax/ElevenLabs voice clone）或合适 TTS 音色。
- **路线已定：自建（端脑云 GPU）+ HeyGem** —— 用户实测 HeyGem 唇形对齐好；端脑云有现成集成方案，部署省事。（SadTalker/EchoMimic/Wav2Lip 作备选）

### 关键协同（终极降本）
既然数字人走端脑云 GPU 自建，**首尾帧主体动画也放同一台 GPU 跑 ComfyUI（Wan 2.2 等开源、支持首尾帧）**：
- 一台 GPU 同时干"数字人对口型 + 首尾帧主体动画"，**P1 视频 + P3 数字人 的 API 钱一起省**（只剩 GPU 租金/电费）。
- 这是用户"彻底省成本只能租算力跑 ComfyUI"判断的落地形态。
- 架构留好"出片后端"抽象：`API(Kie) ↔ 自建(端脑云 ComfyUI)` 可切，前面流程(归并/配音/拼接/字幕)不动。

## 五条件

### 范围
口播类脚本 → 竖屏成片：首尾帧动画主体 + TTS 配音 + 动态字幕（P1-P2）；数字人右下角口播（P3）。

### 输入 / 输出
- 输入：已确认口播文案 + 视觉风格 + 平台 +（可选）参考图 +（P3）用户形象素材/声音样本。
- 输出：MP4（9:16），长度=口播；附分镜清单 + 成本明细。

### 约束（铁律）
- 脚本 > 大模型：分镜归并、字幕、拼接、画中画合成全用脚本/ffmpeg。
- 能不花钱不花钱：付费只在"首尾帧 AI 动画"和"数字人"，拼接/字幕/合成本地 0 元。
- 稳定 > 花哨：拼接在 122 本地 ffmpeg，失败如实报。
- UI 不露黑话：只说"配音/出片/数字人/档位"，不出现模型名/服务商。
- 成本可配（config.js 一处）：模型、单价、目标分镜数、时长。

### 验收（含反 PPT 硬红线）
- [ ] ~30-45s 口播 → 单条成片，长度=口播 ±2s。
- [ ] 主体画面**真动**（首尾帧补间或 AI 视频），**无翻页 PPT、无整页静图硬切**。
- [ ] 前 3 秒钩子是动态画面。
- [ ] 画面风格全片统一，紧扣主题。
- [ ] 字幕与人声对齐。
- [ ]（P3）右下角数字人对口型，口型与音轨基本同步。
- [ ] 出片前显示成本 + 时长预估；中途刷新可断点续接。

### 非目标（本期）
- 复杂特效、多轨混音。
- 自建 ComfyUI 出片（phase-2 降本，架构留好"出片后端"可换 API↔端脑云）。

## 架构

```
口播文案
 → ① 归并分镜(默认5, 脚本)                         [P1]
 → ② 每镜出 首帧图 + 尾帧图(锁风格, 紧扣该beat)      [P1]
 → ③ 首尾帧补间 → 主体动画片段(AI, 真动)            [P1]
 → ④ TTS 配音(全文) → 主音轨 + 句级字幕时间轴        [P2]
 → ⑤ ffmpeg 拼接: 动画序列 + 音轨 + 烧动态字幕 → 成片 [P2]
 → ⑥ 数字人(本人形象, 对口型音轨) → 右下角画中画合成   [P3]
```

## 成本模型（你的真价 0.036 元/点，待按后台校准）
- 关键帧图：5 镜 × 2 帧(首+尾) × 3 点 = 30 点 ≈ 1 元
- 首尾帧动画：5 段 ×（待定模型单价）。若 Seedance fast 480p 6s≈93点/段→5 段≈465点≈17元（**贵，需找按次模型**）；若有按次首尾帧模型（如 30 点/段）→5 段≈150点≈5.4元。
- TTS：几毛；拼接/字幕/合成：本地 0 元。
- 数字人：phase-3 另算。
- **目标：一条 30s 口播 ≤ 6 元**（取决于首尾帧模型单价 → 先确认）。

## 分步实现
1. **P1a 归并分镜**（config 已加 `VIDEO_TARGET_SHOTS=5`）：改 `buildVideoShots` 归并，不再一句一镜。→ 立即砍成本。
2. **P1b 首尾帧出图 + 补间**：每镜出首/尾两帧；选定"便宜的首尾帧模型"（**先按你后台确认**）。
3. **P2 TTS 配音 + ffmpeg 拼接 + 字幕**：122 装 ffmpeg + 中文字体；MiniMax TTS 代理。
4. **P3 数字人右下角口播**：声音克隆 + 对口型 + 画中画合成（选型另定）。
