# Spec:内容系统化补全 v1(选题判断字段 + 传播目的 + 复盘3问)· 2026-06-26

来源:对标外部案例《用 Codex 做视频二创项目落地》(F:\Longka Wiki\…\用 Codex 做视频二创项目落地案例.md),思路与我们一致,补我们闭环里真实存在的缺口。不依赖 mizan 真实答案,现在就能做。

**① 名字**:content-system-completion-v1

**② 触发词**:选题判断字段 / 传播目的 / 复盘表 / 对标库补字段

**③ 范围(这次做什么)**

1. **选题候选/对标层补 4 个"判断字段"**(我们四样料都已有,只是没绑到选题这一行):
   - `target_persona` 适合人群 —— 挂 `personas_mizan.json` 的 persona id/标签
   - `fit_formats` 适合改成哪种短视频 —— 总图 6 形态 + 下方 5 传播目的(数组)
   - `risk_flags` 风险提醒 —— 调 `compliance_scan.py` 给的风险等级/类型(导流/承诺/高危行业)
   - `ref_assets` 参考素材 —— 关联 `assets.db` 里按标签命中的 norm_clip id(给配镜起点)
   落点:`longka_benchmark`(对标拆解层)新增列(additive `alter table add column if not exists`,不破坏现网);选题候选对象同步带这 4 字段。

2. **「5 种传播目的」形态层**(借自案例,叠在现有"内容角度"之上):
   - 枚举:`曝光型`(反常识开头)/`收藏型`(步骤清单)/`评论型`(抛争议)/`转化型`(落到产品)/`人设型`(过来人判断)
   - 用法:同一选题 × 脚本框架库(揭价格/盘渠道/选品/算账)× 5 目的 → 让批量 N 条**话题不同 + 目的不同**,每条有明确 KPI 意图(目的→主指标:曝光=完播、收藏=收藏率、评论=评论率、转化=线索、人设=关注)。
   - 落点:`script_frames.py` 框架加 `purpose` 维度;编排器 `build_scripts.py` 按 目的×框架 组合出稿。

3. **复盘表 + 固定 3 问**(把"我觉得不错"变成数据判断,喂数据复利护城河):
   - 新表 `longka_review`:title/platform/published_at/选题类型/开头方式/时长/播放/完播率/赞/收藏/评论/转发/评论关键词/下次优化。
   - 固定 3 问(每周喂数据,脚本聚合,非 LLM 现场推理):①哪类**选题**更容易被点开(按 选题类型/目的 看 CTR)②哪类**结构**更容易被看完(按 开头方式/时长 看完播)③哪类**观点**更容易引评论收藏(按 目的/争议度 看 评论+收藏率)。
   - 落点:`collector-hub.mjs` ensureTables 加 `longka_review`;聚合脚本 `tools/review_rollup.py`。

**④ 禁止范围(这次不做)**
- 不碰 mizan 文案内容(等真实答案);不改既有 samples/comment_insights 表结构(只新增,不动列);
- 不做前端复盘 UI(先把表+聚合脚本跑通,UI 另立);
- 不退回"给客户对话框"——这些都沉后端工作流,客户零技术(守产品化北极星);
- 不依赖付费 API(persona/形态/目的=脚本枚举;风险=已有 compliance_scan;聚合=脚本)。

**⑤ 验收标准**
- `longka_benchmark` 多出 4 列、`alter` 在现网幂等不报错、小妹无感;
- 给一条选题,能自动带出:适合人群(persona)、建议形态+目的、风险等级、3 条参考素材(assets.db 命中);
- `build_scripts.py` 能按 目的×框架 对同一选题出 ≥5 条**目的不同**的稿;
- `longka_review` 建表 + `review_rollup.py` 喂样例数据能答出固定 3 问(带数字)。
