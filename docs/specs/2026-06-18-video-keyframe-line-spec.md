# 视频线 · 首尾关键帧出图 + 图生视频 Spec

- 日期：2026-06-18（用户口述原理 + 爬 Kie 文档印证后立 spec）
- 类型：开发 spec（重做视频线的"出图→出片"链路）
- 依据：用户定调"图生视频必须用首尾关键帧、出图阶段就得为视频出关键帧";Kie 文档证实 Seedance/Wan 图生视频支持 `first_frame_url` + `last_frame_url`。

## 背景（为什么重做）

当前视频片段(B 路)拿**通用模板图文卡**当首帧出片 → 图上文字是"跑出结果/服务谁发什么"这类**写死模板**,跟用户的脚本完全不搭(实测：抖音脚本讲"内容人怎么应对DeepSeek融资",图上却是"震惊!估值50亿(数字还错)+跑出结果")。**源头(帧)就错,视频必错。**

核心认知(用户)：**AI 图生视频 = 关键帧驱动**。模型在**首帧→尾帧之间补运动**。所以：
1. 关键帧就是视频的源头真相,内容必须对(匹配脚本/分镜);
2. **出图阶段就得直接出"分镜的首尾关键帧"**,而不是出一张通用图再硬拿去当帧。出图与出片不能脱节。

## 1. 名字
视频线 · 首尾关键帧流水线

## 2. 范围（关键帧驱动的出片流程）

```
确认的脚本
 → ① 切分镜（按"拍"切成 N 镜，每镜一句话/一个画面意图）
 → ② 出图阶段＝出关键帧（不是出小红书图文卡）：
       每镜出 首帧（必）+ 尾帧（可选）；
       画面内容严格来自该镜脚本（不用通用模板套话）；
       风格＝所选配图风格；首尾帧同主体同画风，只变动作/状态。
 → ③ 图生视频：每镜 first_frame_url(+last_frame_url) 喂 Kie 图生视频 → 一段
 → ④ 拼接：各镜段按脚本顺序拼成成片
```

## 3. 技术落点（已被 Kie 文档证实）

- **kie-video.mjs**：现仅发 `first_frame_url`，加 **`last_frame_url`**（Seedance/Wan 的"首尾帧"模式）；模型默认 `bytedance/seedance-2-fast`，可 env 切 `wan/2-7-image-to-video`。
- **关键帧出图**：复用 kie-image，但 brief **从该镜脚本提炼**（杀掉 `extractVisualTopicSignals` 的"跑出结果/服务谁发什么"等写死模板）；尾帧 = 首帧的"结束状态"（同主体、动作推进）。
- **脚本切分镜**：脚本 → N 镜，每镜给 {首帧画面描述, 尾帧画面描述, 运镜/动作}。
- **数字/事实不臆改**：钩子/画面文字忠实脚本（不把 500 亿写成 50 亿、不硬加"震惊"）。

## 4. 禁止范围
- ❌ 不再拿小红书通用图文卡当视频首帧。
- ❌ 出图阶段不能脱离脚本/分镜（不用通用模板兜底词）。
- ❌ 不裸烧钱：先 static/单镜验证链路，再放量。
- ❌ 客户端不暴露 Kie/Seedance/Wan/首尾帧等内部名（大白话：让画面动起来/做视频）。

## 5. 验收标准
- 给一条脚本 → 切出分镜 → 每镜出的关键帧**文字/画面跟该镜脚本一致**（不出现"跑出结果"这类与内容无关的模板词）；
- 首尾帧同主体同画风；图生视频在首→尾间补运动；拼成成片与脚本对得上；
- 数字/事实与脚本一致；客户端零黑话。
