## 背景

视觉/封面相关改造此前在对话里"方向对齐就动手"，已上线但裸奔无 spec（违反 SPEC-first）。
本 spec 把既成事实固化、划清边界，并把本期要新增的人像对标封面纳入范围。

封面 = 冷启动 CTR 第一闸；"拿配套内页第一页当封面"是错的，封面必须独立生成。
内容号默认零上传（小妹不传人像/对标图），人像对标只服务个人形象/IP 号。

## 3. 范围（可执行动作）

1. **出图主力切 Kie gpt-image-2**（`kie-image.mjs`）：
   - createTask → 轮询 recordInfo；`/api/xhs-cards/generate-xiaohei/start` + `/status` 走 Kie 分支。
   - KIE_API_KEY 在 env；`kieEnabled()` 为假时回落 43 代理。
   - **Kie 慢自动兜底 43**：Kie 任务跑 > `KIE_FALLBACK_MS`（默认 100s）仍无图 → status 处理里自动用同样的卡起一个 43 任务，谁先出图用谁（Kie 健康 ~85s 走 Kie；真卡住才转 43）。原因：Kie 排队拥堵时出图可达 5 分钟，是外部不可控因素（实测纯英文短提示词也要 4 分钟+，与提示词长短无关）。
   - **43 封面 passthrough**：43 的 `buildLongkaVisualPrompt` 加了 `LONGKA_COVER_PASSTHROUGH` 分支——`card.role==='cover'` 且带 `imagePrompt` 时原样用该提示词（+保留 43 负向约束），使 43 兜底出的是和 Kie 同款单焦点实拍封面，而非 43 默认的归藏多格资讯卡。仅动 43 longka 路径，不碰色彩小程序。
2. **内容驱动独立封面 `cover-from-content`**：
   - 从已写正文提炼诚实钩子（数字/痛点/提问，**不焦虑**）。
   - 产出真实拍摄感、米金大字、无人脸、单焦点封面提示词 → Kie 出单张封面。
   - 制作中心（step 10）小红书有独立"生成封面"面板（`renderCoverPanel` + `generateCoverFromContent`）。
3. **账号默认风格锁**：
   - `state.visualStyleByLine` 按行锁风格（归藏暖金不乱跳）。
   - `recommendVisualRouteClean` 中文关键词路由修复（避坑/痛点→小黑，流程/步骤→归藏 等）。
4. **per-platform 画幅**：小红书 3:4、公众号 3:2、朋友圈 1:1。
5. **人像对标封面 `cover-replicator` 接 UI（本期新增）**：
   - 个人形象/IP 号赛道，支持上传人像 + 对标图。
   - 深度复刻结构（画幅/标题层级/字体气质/人物位置/背景氛围）+ 人像一致锁定。
   - 接进工作台独立路径，与内容号封面互不混。

## 4. 禁止范围

- **不做图片持久化存储**（Kie 临时 URL ~24h 过期落地）——留单独 spec，blocked 在火山云 OSS key。
- **不改 43 色彩搭配卡片小程序的出图代码**（只动 43 的 longka 封面路径加 passthrough；43 作 Kie 关闭**或太慢**时的兜底渲染器）。
- **不换 gpt-image-2 模型**。
- 内容号（私校/美容/女性成长）封面**零上传**，不要求小妹传人像/对标图；上传仅限 cover-replicator 个人形象号路径。
- 内容号封面**不抄对标帖封面**（雷同），只从本篇正文提炼。
- 不动已认可的前端模块拆分结构。

## 5. 验收标准

1. 小红书制作中心确认文案后出现"小红书封面（独立生成）"面板，点"生成封面" → Kie 出一张真实拍摄感、米金大字、无脸封面 + 2-3 个诚实钩子候选。
2. 同账号连续做多篇，封面/配图风格稳定在账号默认风格，不随机跳成小黑。
3. 三平台画幅正确：小红书 3:4 / 公众号 3:2 / 朋友圈 1:1。
4. 个人形象号路径：上传人像 + 对标图 → 产出人像一致、复刻对标结构的封面。
5. 全程走 Kie（KIE_API_KEY 在 env）；43 色彩小程序出图不受影响、仍正常。
