# 三天稳定版计划 Spec

## 1. 名字

三天稳定版计划

本 spec 也可按当前执行节奏称为：10 小时冲刺版。

## 2. 触发词

后续用户说以下内容时，必须先读本 spec：

- 按三天稳定版
- 三天稳定版 Day1
- 明天要给投资人看
- 小红书图文成稿样板
- 文案不丢稿
- 122 验收
- 10 小时冲刺版

## 3. 背景

当前 Longka AI Native 已有内容工作台、采集能力、内容资产方向、文案生成和部分图文/生图能力。

但当前问题是：

- 文案迭代不稳定，继续优化可能只追加建议。
- 文案版本容易丢，不能可靠回看、恢复、确认。
- 内容资产、标题库、结构库、客户问题库还没有稳定调用起来。
- 投资人演示需要看到 Longka 不只是写文案，还能做出小红书图文成稿。
- 本地跑通不够，最终必须以 122 公网页面为验收环境。

本 spec 的目标是把当前半成品推进到一个能给团队和投资人看的稳定样板。

## 4. 范围

### Day1：文案版本管理器

必须做到：

- 文案生成后自动保存第 1 版。
- 点击继续优化后，调用 AI 基于当前正文真实改写。
- AI 改写必须接收当前标题、当前完整正文、当前版本、优化方向、体检反馈和源头素材。
- 每一版可回看。
- 每一版可恢复。
- 每一版可确认。
- 确认后不丢稿。
- 再次优化时取消旧确认状态，避免误用旧稿。
- 不允许继续优化只在文末追加建议。

### Day2：内容资产调用 + 图文样板

必须做到：

- 默认使用已采集 X 资产 + Longka 自身方法论，保证演示闭环稳定。
- 如果素材质量不够，可重新换几个 X 账号采集少量高质量帖子。
- 不把实时采集作为投资人演示的必经前置。
- 主题优先做：AI 自媒体 / 内容资产库 / AI 写作没流量。
- 用户确认文案后，生成 5 张小红书图文卡片预览：
  1. 封面
  2. 痛点 / 问题卡
  3. 判断 / 方法卡
  4. 步骤 / 清单卡
  5. 行动 / 咨询卡
- 页面先稳定展示卡片预览。
- 接入“生成 PNG 成稿”按钮。
- 如果真实生图成功，展示 PNG 成稿。
- 如果真实生图失败，页面卡片预览仍可用于演示，但必须显示失败原因，不能伪装成真实出图。

### Day3：今日工作台 SOP + 122 验收

必须做到：

- 页面引导清楚。
- 当前步骤清楚。
- 按钮含义清楚。
- 等待状态清楚。
- 空数据提示清楚。
- 失败原因清楚。
- 本地验证后同步 122。
- 最终以 122 公网地址验收：

```text
http://122.51.218.154/ai-native-v2/workbench-v2.html
```

## 5. 禁止范围

本轮不做：

- 完整商业图文工厂。
- 完整视频闭环。
- 小妹本地包打包。
- 所有平台采集打通。
- 标题库大规模重构。
- 内容资产库完整训练体系。
- 本地 Qwen 3B 训练。
- 发布复盘模块完整实现。

禁止行为：

- 不用假数据冒充真实采集。
- 不把历史资产说成新采集。
- 不把本地验证当成 122 验收。
- 不为了图文样板重构整个工作台。
- 不把 Day1 文案版本任务扩展到采集、母题、标题库。
- 不在真实生图失败时用假图片冒充成功。

## 6. 验收标准

### Day1 验收

- 生成正文后出现第 1 版。
- 点击继续优化后新增第 2 版。
- 第 1 版仍可回看。
- 第 1 版可恢复。
- 第 2 版可确认。
- 确认后页面显示已确认状态。
- 再次优化会取消旧确认状态。
- 优化结果不是只在文末追加建议。

### Day2 验收

- 使用 AI 自媒体 / 内容资产库 / AI 写作没流量主题生成一篇确认文案。
- 确认文案后生成 5 张小红书图文卡片预览。
- 卡片不是纯文字堆叠，有封面、问题、方法、清单、行动入口结构。
- 可以点击生成 PNG 成稿。
- PNG 成功则展示图片。
- PNG 失败则展示明确失败原因，并保留网页卡片预览。

### Day3 验收

- 本地能跑通一次：

```text
资产/素材
-> 文案生成
-> 版本优化
-> 确认版本
-> 图文卡片预览
-> 尝试生成 PNG
```

- 122 能跑通一次投资人演示流程。
- 页面不会让用户迷路。
- 等待和失败状态都可见。

## 7. 时间目标

用户判断：这些任务在连续开发和测试配合下，10 小时内应当可以交付一个投资人可看的样板。

执行策略：

- 优先交付可演示闭环。
- Day1 文案版本管理器优先。
- 图文样板以网页卡片预览为稳定兜底。
- 真实 PNG 出图作为增强能力接入。
- 122 验收必须做，但如果真实外部接口失败，要明确记录失败原因。

## 8. 开发前强制复述

每次按本 spec 开发前，AI 必须复述：

```text
我已读取三天稳定版计划。
本轮目标是：文案版本稳定、内容资产调用、图文样板、122 验收。
当前只做指定 Day 的任务。
不扩大到视频、小妹包、完整训练体系。
验收以页面可操作和 122 公网验证为准。
```
