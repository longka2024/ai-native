# Spec:私校知识库 + 二次改写注入 · 2026-06-26

**背景**:私校线现在文案=对标爆款的"壳",缺真干货。把权威资料(如圣乔治招生白皮书)做成知识库,在小妹现有流程的**第6步「二次改写」**自动注入真实知识点 → 文案从"空壳"升级成"壳+官网没有的内部干货"(增量价值 + 天然合规:过来人分享真知识,非招揽承诺)。

**① 名字**:private-school-knowledge-base

**② 触发词**:知识库 / 白皮书 / 私校知识 / 知识卡 / 改写注入

**③ 范围(这次做什么)**

1. **知识表 `longka_knowledge`**(collector-hub ensureTables,additive):
   `id / workspace(私校留学) / school(圣乔治…) / grade(KG/G4/G6/G8/G9/G11/通用) / category(学校概况/名额/时间轴/评估/内部洞察/常见错误/准备建议/Checklist) / point(知识点文本) / tags(jsonb 关键词) / source(白皮书 Part X) / created_at`。

2. **`tools/ingest_knowledge.py`**:把白皮书 HTML **脚本拆成知识卡**(按 Part/Chapter/年级/EDinsight 段落切分),自动打标(school/grade/category/tags)→ 输出 `knowledge_<school>.json` → 灌 `longka_knowledge`。可重复用于以后更多学校的资料。

3. **第6步改写注入钩子**(rewrite-engine.js,复用现有真实料注入范式):
   - 私校 workspace 改写时,按选题(school/grade/主题关键词)**检索相关知识卡**(同 `filterPainsRelevantToTopic` 的不跑题筛选);
   - 注入改写 prompt 当真实料,带**铁律3 硬约束**(必须用、来源可追溯、绝不编)+ **不跑题**(泛选题不灌具体学校细节);
   - 小妹操作不变、无感(产品化北极星)。

**④ 禁止范围(这次不做)**
- 知识卡**不进小红书爆款选题库**(两库分工:爆款库=壳/结构,知识库=填真干货),不污染选题搜索;
- 不动小妹现有 1–5 步、7–8 步的操作流程,只在第6步后台注入;
- 不编知识(铁律3):知识卡只来自白皮书原文,带 source 可追溯;泛选题宁可不注入也不硬塞(不跑题);
- 不做知识库管理 UI(先把表+入库+注入跑通,UI 另立);不烧付费 API(纯脚本解析 + 现有 DeepSeek 改写)。

**⑤ 验收标准**
- 这份圣乔治白皮书拆出 ≥30 条知识卡,标签准(年级/类别对得上),带 source;
- 给一个私校选题(如"圣乔治 Grade 8 怎么准备"),能检索出**相关**知识卡(G8名额/常见错误/招生官看什么),泛选题不误注入;
- 改写后的文案**带白皮书真实知识点**(可追溯 source),且不跑题、不编;
- `longka_knowledge` 建表 additive 不破现网;小妹流程零改动。
