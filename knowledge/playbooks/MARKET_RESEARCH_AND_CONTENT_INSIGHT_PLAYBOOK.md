# 市场调研与内容洞察工作法

这份文档记录一套长期复用的方法：如何把专业数据工具、Agent Reach、人工判断和产品改造结合起来，用于新项目的市场调研、内容选题、页面文案和转化设计。

## 1. 核心判断

不要把 Agent Reach 当成专业数据平台。

它更像是“人工调研自动化工具”：可以帮我们快速搜索、读取、整理真实内容样本，但它不等于全量市场数据。

专业数据工具和 Agent Reach 的分工是：

- 专业数据工具：找矿区。
- Agent Reach：挖样本。
- Codex/Agent：把样本转成产品动作、内容动作和运营动作。

## 2. 专业数据工具适合做什么

专业数据工具适合回答这些问题：

- 哪些关键词热？
- 哪些内容最近趋势上升？
- 哪些账号在这个领域做得好？
- 哪些笔记互动率高？
- 哪些赛道有商业机会？
- 哪些达人、话题、标签值得跟踪？

常见工具类型：

- 小红书数据工具
- 抖音/短视频数据工具
- 达人投放平台
- 关键词趋势工具
- 广告投放后台
- 行业榜单工具

专业工具的价值是“方向判断”和“流量地图”。

## 3. Agent Reach 适合做什么

Agent Reach 适合回答这些问题：

- 用户具体怎么说？
- 博主标题具体怎么写？
- 封面用什么钩子？
- 评论里用户真实焦虑是什么？
- 竞品怎么描述服务？
- 哪些词能让普通人看懂？
- 页面和视频脚本应该怎么接住这些需求？

Agent Reach 的价值是“真实语料”和“转化洞察”。

它不负责证明市场全量规模，它负责把样本变成可执行动作。

## 4. 两者怎么配合

推荐工作流：

1. 用专业数据工具找关键词、爆文、竞品账号、趋势方向。
2. 把这些关键词、链接、账号交给 Agent Reach。
3. Agent Reach 抓取标题、正文、评论、互动数据和图片线索。
4. Codex 整理出高频痛点、成交钩子、内容结构和用户顾虑。
5. 把洞察转成：
   - 首页文案
   - 小程序介绍
   - 视频脚本
   - 小红书标题
   - 封面文字
   - 客服话术
   - 产品流程改造
   - 付费转化提示

一句话：

> 专业工具找方向，Agent Reach 抓语料，Codex 负责变成产品和营销动作。

## 5. 没有专业数据工具时怎么做

没有专业工具也能先跑低成本版本。

做法是建立关键词池，不只搜一个词。

例如个人色彩/形象报告项目：

直接需求词：

- 个人色彩分析
- 四季型色彩测试
- 形象设计
- 风格测试
- 穿搭诊断

痛点词：

- 不会穿搭怎么办
- 黄黑皮显白
- 冷暖皮测试
- 显老怎么办
- 土气感怎么改善

结果词：

- 普通女生变美
- 素人改造
- 变美思路
- 氛围感穿搭
- 提升气质

付费意向词：

- 线上形象分析
- 线上穿搭诊断
- 色彩诊断多少钱
- 个人形象顾问
- 妆发穿搭建议

每个词至少抓：

- 最新内容
- 热门内容
- 评论多的内容
- 标题强的内容
- 明显有商业转化的内容

## 6. 如何判断样本有没有价值

不要只看点赞。

更有价值的样本包括：

- 点赞高：说明内容形式有传播力。
- 评论多：说明用户有问题、有焦虑、有争议。
- 收藏高：说明用户觉得有用，可能有长期需求。
- 标题强：适合学习表达方式。
- 评论里有人问价格/怎么测/准不准：说明接近成交。
- 作者有明确引流动作：说明可能是商业内容。

对商业项目最有价值的不是“热闹”，而是“能转化的用户问题”。

## 7. 输出物标准

调研最后不能只输出一堆链接。

必须输出这些可用资产：

- 关键词池
- 爆款标题模板
- 封面文案方向
- 用户痛点清单
- 用户顾虑清单
- 成交钩子
- 竞品表达方式
- 我们应避免的表述
- 首页/落地页文案建议
- 视频脚本方向
- 客服答疑话术
- 产品流程改造建议

## 8. 用在个人色彩项目上的原则

这个项目不能只卖“生成图片”。

它应该卖：

- 我更适合什么颜色
- 我怎样看起来更有气质
- 我怎样避免穿错颜色
- 我如何知道自己的妆发和穿搭方向
- 我能不能先看到一个可信的试看结果
- 付费后能不能拿到完整清晰的报告图

所以调研时重点看：

- 用户为什么想测色彩
- 用户怕什么
- 用户怎样判断准不准
- 用户愿意为什么结果付费
- 用户会把什么图发朋友圈
- 用户觉得什么文案一看就懂

## 9. 长期沉淀方式

每次调研后，要把结果沉淀到项目资产里：

- `docs/MARKET_RESEARCH.md`
- `docs/CONTENT_STRATEGY.md`
- `docs/CUSTOMER_OBJECTIONS.md`
- `docs/CONVERSION_COPY.md`
- `docs/COMPETITOR_NOTES.md`

如果某套方法可复用到其他项目，就沉淀到全局长期基建：

- `E:\Codex\MARKET_RESEARCH_AND_CONTENT_INSIGHT_PLAYBOOK.md`

## 10. Agent 操作规则

以后用户自然语言说：

- “帮我搜小红书这个赛道”
- “帮我分析竞品笔记”
- “帮我找用户痛点”
- “帮我拆爆款标题”
- “帮我看这个项目怎么推广”

Agent 应该优先按这个流程执行：

1. 明确项目目标和目标用户。
2. 建关键词池。
3. 用 Agent Reach 抓样本。
4. 按标题、正文、评论、互动、转化动作分类。
5. 输出可执行的产品和营销建议。

不要只返回搜索结果。
