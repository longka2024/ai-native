# Product Result First Principle

这是 Longka 做产品和行业 AI 助理 U 盘的第一原则。

## 1. 客户要结果，不要技术

客户通常不关心：

- 用了什么模型
- 接了什么 MCP
- 抓了多少数据
- 后台怎么调度
- prompt 多复杂
- 技术链路多先进

客户关心：

- 我能得到什么结果？
- 能不能帮我省时间？
- 能不能帮我获客？
- 能不能帮我赚钱？
- 能不能让我看起来更专业？
- 我下一步该点哪里？
- 我能不能马上拿去用？

## 2. 产品展示顺序

正确顺序：

```text
结果
-> 下一步动作
-> 样例
-> 可信证据
-> 过程说明
-> 技术细节
```

错误顺序：

```text
技术原理
-> 数据来源
-> 模型能力
-> 流程说明
-> 最后才展示结果
```

## 3. 应用到行业 AI 助理 U 盘

不要卖：

```text
内置 Agent Reach、Exa、MCP、Remotion、skills、脚本。
```

要卖：

```text
插上后，帮你每天找选题、拆爆款、写脚本、生成图文、做视频、整理客户话术。
```

律师版卖：

```text
每天给你整理本地客户关心的法律问题，生成普法短视频脚本和咨询转化话术。
```

家装版卖：

```text
每天帮你找小红书/抖音装修爆款，生成避坑图文、客户跟进话术和案例视频脚本。
```

美业/形象设计版卖：

```text
每天帮你找女性用户关心的变美问题，生成小红书标题、封面、对比图文和短视频脚本。
```

私域增强版卖：

```text
读取你自己授权的客户群聊天，总结今天客户最关心的问题，生成回复话术和明天可发的内容选题。
```

## 4. 应用到色彩报告产品

不要说：

```text
我们使用 AI 图像生成、多平台内容情报和自动化报告系统。
```

要说：

```text
上传一张照片，先看你适合什么颜色、妆发和穿搭方向。满意后再解锁完整形象报告。
```

## 5. 应用到研究简报

不要输出：

```text
这里有 20 条搜索结果。
```

要输出：

```text
我帮你筛出了 8 条最值得看的资料，分成入门、进阶、实战和避坑。最值得先看的是这 3 条，因为它们能直接变成你的内容 SOP。
```

## 6. Longka 硬规则

任何产品、页面、报告、视频、行业包，都先问：

1. 客户最终想拿到什么？
2. 这个结果能不能马上展示？
3. 下一步动作是否清楚？
4. 技术细节是否干扰了客户理解？
5. 数据是否已经被转化成判断和行动？
