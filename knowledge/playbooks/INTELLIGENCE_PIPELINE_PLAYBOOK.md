# 情报采集与任务编排工作法

这份文档用于沉淀“24 小时情报员”式工作流：让数据自动进入决策流程，而不是靠人每天手动搜索、复制、整理。

参考来源：

- 本地剪藏：`F:\Longka Wiki\龙咖Wiki\Clippings\别再手动搜资料了，我用 Hermes + XCrawl 搭了个 24 小时情报员.md`
- 原文链接：`https://x.com/wsl8297/status/2054798253955375388`

## 1. 核心思想

真正有价值的不是“抓网页”本身，而是：

> 让数据自动进入我们的决策流程。

也就是说，系统不只是帮我们搜资料，而是持续做这几件事：

1. 知道要盯什么。
2. 定时或按需去抓。
3. 抓完自动整理。
4. 判断哪些值得提醒。
5. 输出成报告、表格、选题、风险提醒或待办。

## 2. 角色分工

### 专业数据工具

负责找方向：

- 关键词热度
- 趋势变化
- 达人/账号榜单
- 爆款内容
- 行业数据
- 广告/投放线索

### Agent Reach

负责轻量调研和多平台样本读取：

- 小红书笔记
- 网页
- RSS
- 视频字幕
- GitHub
- 其他已接入平台

适合随问随查、样本分析、竞品内容拆解。

### XCrawl

负责复杂数据获取：

- 任意网页抓取
- JavaScript 渲染页面
- 复杂列表页
- 竞品官网
- 搜索结果页
- 商品页/评论页
- 需要代理或更稳定抓取的页面

XCrawl 像“数据管道”。

### Hermes Agent

负责任务编排：

- 判断抓什么
- 判断什么时候抓
- 判断抓完怎么整理
- 判断哪些变化值得提醒
- 输出日报/周报/表格/知识库/推送

Hermes Agent 像“调度员”。

### Codex

负责把情报变成动作：

- 产品改造
- 页面文案
- 小红书选题
- 视频脚本
- 客服话术
- 后台需求
- 技术任务
- 项目计划

## 3. 总体结构

推荐结构：

```text
数据源 -> 采集工具 -> 任务编排 -> 分析整理 -> 执行动作
```

对应到我们的长期基建：

```text
专业数据工具 -> 找市场方向
Agent Reach -> 抓真实样本
XCrawl -> 抓复杂网页/动态页面
Hermes Agent -> 定时调度和提醒
Codex -> 分析、写文案、改产品、生成任务
```

## 4. 三类典型场景

### 场景 1：行业情报收集自动化

用途：

- 行业调研
- 竞品监控
- 选题收集
- 政策/平台规则跟踪

流程：

1. Hermes 接收关键词或目标网站。
2. XCrawl 或 Agent Reach 抓搜索结果、网页、标题、摘要、发布时间。
3. 系统自动去重、分类。
4. 输出每日简报。
5. Codex 把简报转成选题、产品建议或待办。

价值：

- 不是搜一次就结束。
- 而是每天维护一个动态情报库。

### 场景 2：社媒互动雷达

用途：

- 跟踪 KOL 动态
- 监控竞品账号
- 捕捉热门话题
- 找最佳互动窗口

流程：

1. 设置目标账号和关键词。
2. 抓取推文/笔记/评论/转发/互动数据。
3. Hermes 筛选高价值互动窗口。
4. 自动生成互动建议。
5. 推送到表格、Notion、飞书或 Telegram。

价值：

- 从盲目刷平台，变成精准捕捉机会。

### 场景 3：商品/服务监控

用途：

- 价格监控
- 竞品追踪
- 库存变化
- 评论增长
- 市场反馈

流程：

1. 输入商品链接或类目关键词。
2. 定时抓取价格、评分、评论数、库存、页面变化。
3. 对比历史数据。
4. 异常变化触发提醒。
5. 输出竞品变化报告。

价值：

- 把网页变化变成可执行商业信号。

## 5. 用在个人色彩项目上

最小闭环先不要做太大。

第一版可以做：

```text
小红书关键词每日监控 -> Markdown 简报 -> 选题/文案建议
```

关键词池：

- 个人色彩分析
- 四季型色彩测试
- 黄黑皮显白
- 冷暖皮测试
- 素人改造
- 线上形象分析
- 穿搭诊断
- 妆发穿搭建议

每天抓取：

- 最新笔记
- 热门笔记
- 标题
- 作者
- 点赞/收藏/评论
- 正文
- 评论里的用户问题

日报输出：

- 今日高频痛点
- 值得模仿的标题
- 值得学习的封面表达
- 用户担心的问题
- 可以拍的视频选题
- 可以改的首页文案
- 可以优化的付款/报告流程

## 6. 什么时候需要 XCrawl / Hermes

不要一开始就上复杂系统。

先用 Agent Reach 做人工增强调研，当出现这些情况再引入 XCrawl/Hermes：

- 每天都要重复抓同一批关键词。
- 需要定时日报。
- 需要监控竞品网页变化。
- 需要抓动态渲染页面。
- 需要把结果推送到表格/知识库/群。
- 人工搜索已经明显跟不上节奏。

## 7. 最小实现路线

第一阶段：手动触发

```text
用户自然语言提出调研需求 -> Codex 调 Agent Reach -> 输出分析报告
```

第二阶段：半自动

```text
固定关键词池 -> 一键脚本抓取 -> Markdown 报告
```

第三阶段：自动化

```text
定时任务 -> 抓取 -> 去重 -> 分析 -> 推送日报
```

第四阶段：中台化

```text
多项目关键词池 -> 多平台采集 -> 情报库 -> 任务分发 -> 产品/运营/内容动作
```

## 8. 长期资产沉淀

每个项目都应该沉淀：

- 关键词池
- 竞品账号列表
- 爆款内容库
- 用户痛点库
- 评论问题库
- 标题模板库
- 封面文案库
- 转化话术库
- 每日/每周情报报告

这些资产比一次性搜索结果更重要。

## 9. Agent 工作规则

以后用户说：

- “帮我做行业情报”
- “帮我监控竞品”
- “帮我搜今天小红书热点”
- “帮我看这个赛道最近用户在关心什么”
- “帮我做一个 24 小时情报员”

Agent 不应该只返回链接。

应该按这个顺序工作：

1. 明确目标。
2. 选择数据源。
3. 选择采集工具。
4. 抓取样本。
5. 去重分类。
6. 提炼趋势/机会/风险。
7. 输出业务动作。
8. 把有价值的规则沉淀到长期基建。
