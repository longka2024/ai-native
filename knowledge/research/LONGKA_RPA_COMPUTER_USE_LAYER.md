# Longka RPA / Computer Use Execution Layer

目标：把行业工作人员一天的任务从“生成建议”推进到“半自动执行”，覆盖选题、生成、排版、渲染、发布、跟进。

## 1. 定位

Longka 不是只给建议，而是按一天工作流带着用户完成：

```text
看今日需求
-> 定选题
-> 生成脚本
-> 生成视频/图文
-> 发布到平台
-> 监控评论/私信
-> 生成回复
-> 复盘明日选题
```

执行层可以由三类能力组成：

- API/CLI：稳定、可批量、优先使用。
- RPA/浏览器自动化：平台没有 API 时使用。
- Computer Use：需要看屏幕、点按钮、处理复杂网页时使用。

## 2. 分层原则

### 优先级 1：API / CLI

适合：

- Agent Reach 搜索小红书/公众号
- wx-cli 读取本地微信群
- Remotion 渲染视频
- FFmpeg 后处理
- 云端队列
- 文件整理

优点：

- 稳定
- 快
- 可记录
- 便于自动化

### 优先级 2：RPA / Playwright / 浏览器自动化

适合：

- 登录后台
- 填写表单
- 上传素材
- 打开平台发布页面
- 截图验证
- 下载结果

要求：

- 必须有失败检测。
- 必须有人工接管按钮。
- 不能默认操作敏感动作。

### 优先级 3：Computer Use

适合：

- UI 经常变化的平台。
- 需要视觉判断的操作。
- 用户桌面软件。
- 没有 API 和稳定 DOM 的场景。

要求：

- 每一步要有可见计划。
- 发布、付款、删除、授权等动作必须人工确认。
- 输出操作日志。

## 3. 一天工作流自动化

### 09:00 看今日需求

输入：

- 微信群聊天
- 小红书/公众号爆款
- 昨天评论/私信

工具：

- wx-cli
- Agent Reach
- Exa / mcporter

输出：

- 今日需求摘要
- 今日主推选题

### 09:20 生成内容

工具：

- Longka content creative
- Longka video workflow assistant
- ComfyUI / RunningHub / imagegen
- Remotion / FFmpeg

输出：

- 视频草稿
- 小红书图文
- 朋友圈文案
- 私信话术

### 10:30 发布前检查

工具：

- Playwright 截图
- 视频抽帧
- 人工确认

检查：

- 标题是否清楚
- 字幕是否挡脸
- CTA 是否明确
- 是否含敏感词
- 是否误用客户隐私

### 11:00 发布

工具：

- 平台 API，如有
- RPA/浏览器自动化
- Computer Use

要求：

- 发布前必须弹出确认。
- 账号登录由用户自己完成。
- 自动化只填内容和素材，不绕过平台风控。

### 15:00 跟进

工具：

- 平台评论/私信读取，如可用
- 微信群/私聊摘要

输出：

- 评论回复
- 私信成交话术
- 潜在客户列表

### 17:30 复盘

输出：

- 今天发了什么
- 哪些内容值得复用
- 明天选题建议
- 需要老板处理的事项

## 4. 安全边界

必须人工确认的动作：

- 发布内容
- 删除内容
- 付款/退款
- 给客户发送敏感信息
- 上传客户照片到云端
- 使用数字人克隆
- 修改账号设置

可自动执行的动作：

- 搜索
- 摘要
- 生成草稿
- 文件整理
- 视频渲染
- 生成发布文案
- 填充草稿表单

## 5. 产品表达

不要说：

```text
我们集成 RPA、Computer Use、Playwright、CLI。
```

要说：

```text
每天按步骤帮你完成选题、内容、视频、发布准备和客户跟进。
```

## 6. MVP 自动化边界

第一版先做到：

- 自动分析微信群。
- 自动搜索公众号。
- 自动生成选题。
- 自动生成视频脚本。
- 自动生成本地视频草稿。
- 自动生成发布文案。

第一版暂不做：

- 全自动发布。
- 自动私信客户。
- 自动付款/退款。
- 自动跨平台矩阵账号操作。

发布阶段先做“半自动”：

```text
系统生成内容 -> 用户确认 -> 系统打开发布页/准备素材 -> 用户点击最终发布
```
