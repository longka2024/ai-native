# Longka 私域朋友圈成交助手排期

## 产品定位

```text
每天从你的真实微信聊天和群聊里，自动挖出能发朋友圈、能促成交的素材，
生成朋友圈文案、配图、私信承接话术和次日复盘建议。
```

这不是普通朋友圈文案工具，而是：

```text
私域素材雷达 + 朋友圈成交助手
```

## 难度分级

### 最容易：手动导入聊天记录

用户用 `wx-cli` 或手动复制聊天记录，导入系统。

优点：

- 已经验证过 `wx history` 可用。
- 不碰自动登录和自动发布。
- 不容易触发平台风控。
- 48 小时内能做 MVP。

缺点：

- 用户需要多一步导出。
- 不够“傻瓜式”。

结论：

```text
第一阶段必须从这里开始。
```

### 中等：半自动每日素材包

系统每天读取导出的聊天记录，自动生成：

```text
今日可发素材
朋友圈文案
配图提示词 / 图片
私信承接话术
明日选题建议
```

优点：

- 客户能马上看到价值。
- 不依赖复杂 RPA。
- 可以快速卖给私域从业者试用。

结论：

```text
这是第一个真正可演示产品。
```

### 较难：手机端自动读取 / Airtap / RPA

用 Airtap、Computer Use 或手机 Agent 自动进入微信读取内容。

优点：

- 体验更像“自动情报员”。
- 更适合普通小白客户。

难点：

- 微信权限和风控。
- 手机端稳定性。
- 隐私和授权。
- 不同设备适配。

结论：

```text
第二阶段验证，不作为第一版 MVP 的前置条件。
```

### 暂缓：自动发朋友圈

不建议第一阶段做自动发朋友圈。

原因：

- 平台风控风险高。
- 用户也应该人工确认内容。
- 自动发布一旦出错会伤害信任。

正确做法：

```text
生成文案 + 生成配图 + 一键复制 + 保存图片 + 用户手动发
```

## 第一阶段 MVP：48 小时版本

### 目标

做出一个能演示的本地工作流：

```text
导入聊天记录
-> 提取今日素材
-> 分类成交素材
-> 生成 3 条朋友圈
-> 生成配图建议或图片 prompt
-> 生成私信承接话术
-> 输出 HTML 成品报告
```

### 输入

```text
微信聊天导出 txt
微信群聊天导出 txt
用户行业信息
产品/服务说明
客单价
目标客户
```

### 输出

```text
今日素材雷达 HTML
3 条可发朋友圈文案
3 张配图 prompt / 图片占位
3 套私信承接话术
明日继续发什么
```

### 第一版不做

```text
不自动登录微信
不自动抓朋友圈
不自动发朋友圈
不做手机 App
不做复杂后台
```

## 排期

### Day 1：做素材分析 skill

交付：

```text
skills/longka-private-domain-moments/SKILL.md
```

能力：

```text
读取聊天记录
识别客户问题
识别客户顾虑
识别客户反馈
识别成交信号
识别群聊热点
提炼今日可发素材
```

输出结构：

```text
素材类型
原始证据
朋友圈角度
转化作用
适合配图
私信承接建议
```

### Day 2：做 HTML 成品报告

交付：

```text
deliverables/private-domain/moments-daily-pack.html
```

页面模块：

```text
今日私域素材雷达
今日建议发的 3 条朋友圈
每条朋友圈的转化目的
配图建议
私信承接话术
明日选题建议
```

使用 html-anything 思路：

```text
最终给客户看的不是 Markdown，而是可打开、可截图、可复制的 HTML 成品。
```

### Day 3：接 gpt-image2 配图

交付：

```text
每条朋友圈对应 1 张配图
```

优先图片类型：

```text
观点海报
客户问题海报
案例背书图
服务流程图
倒计时预热图
```

原则：

```text
配图是为了提高朋友圈观感和成交信任，不是为了炫技。
```

### Day 4：做手机版交互原型

交付：

```text
一个移动端 HTML 原型
```

流程：

```text
今日聊天素材
-> 推荐朋友圈
-> 点击查看文案
-> 保存配图
-> 复制文案
-> 去微信发布
```

### Day 5：找真实样本试跑

用 1-2 个真实私域样本测试：

```text
形象顾问
知识付费老师
美业老师
```

验证：

```text
提炼出来的素材准不准
朋友圈文案像不像真人
配图是否愿意发
私信话术是否能承接
客户是否愿意付费
```

## 最快能看见结果的版本

最快版本只做这 3 件事：

```text
1. 聊天记录 -> 今日素材分类表
2. 素材分类表 -> 3 条朋友圈文案
3. 朋友圈文案 -> HTML 每日素材包
```

这个版本一天内就能看到效果。

## 优先级结论

先做：

```text
手动导入聊天记录 + 每日朋友圈素材包 HTML
```

再做：

```text
gpt-image2 自动配图
```

然后做：

```text
手机端半自动发布助手
```

最后再考虑：

```text
Airtap / RPA 自动读取
自动发布
多账号矩阵
云端 SaaS
```

第一版不要追求全自动。第一版只要证明：

```text
真实聊天记录可以自动变成客户愿意发的朋友圈成交素材。
```
