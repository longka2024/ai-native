# Longka U 盘产品安全与授权方案

目标：防止代码、模板、API Key、行业工作流被直接复制、倒卖或暴力破解。

## 1. 基本原则

U 盘不是全部资产，只是入口。

正确架构：

```text
U盘/本地端 = 启动器 + UI + 基础模板 + 本地工具调用 + 缓存
云端 = 授权 + 核心行业包 + 核心 prompt/workflow + token gateway + 渲染队列 + 更新
```

不要把以下内容完整放进 U 盘：

- 真实 API Key
- 核心 prompt
- 高价值行业模板全集
- 云端接口密钥
- 客户数据
- 私有源码
- 付费工作流完整实现

## 2. 风险分类

### 2.1 U 盘被复制

风险：

```text
用户买一份，复制给别人。
```

防护：

- 激活码。
- 账号登录。
- 机器码绑定。
- 定期联网校验。
- 关键功能需要云端授权。
- 行业包远程下载。

### 2.2 源码和模板被盗

风险：

```text
别人复制本地文件，拿走 prompt、模板、脚本、工作流。
```

防护：

- 本地只放低敏模板。
- 核心模板服务端下发。
- 高价值 prompt 不写进前端。
- 工作流参数化，不直接暴露完整商业逻辑。
- 本地 JS 可混淆，但不要把安全完全依赖混淆。

### 2.3 API Key 泄露

风险：

```text
你的 key 被拿去刷 token。
```

防护：

- U 盘里绝不放你的真实 key。
- 客户自带 key，或走云端 token gateway。
- 云端做额度、限流、日志。
- 每个用户独立额度。
- 异常请求自动暂停。

### 2.4 暴力破解授权

风险：

```text
破解本地客户端，跳过登录/授权。
```

防护：

- 授权不能只在本地判断。
- 服务端二次校验。
- 授权请求签名。
- 账号 + 机器码 + 订单绑定。
- 可远程吊销。
- 关键行业包和云端功能无授权不返回。

### 2.5 客户隐私泄露

风险：

```text
微信群、客户照片、聊天记录、订单资料泄露。
```

防护：

- 默认本地处理。
- 上传前明确提示。
- 原始聊天默认不上传。
- 自动脱敏。
- 缓存可一键清理。
- 云端只保存必要任务结果。

## 3. MVP 安全方案

第一版不要过度工程化，但必须有基本防线：

```text
1. U盘不内置真实 API Key
2. 本地 .env 由用户自己填写或登录后下发临时 token
3. 启动器需要激活码
4. 激活码绑定机器指纹
5. 高级行业包从云端下载
6. 视频渲染和 token gateway 走云端
7. 输出文件带用户标识水印
8. 日志记录调用次数和异常
```

## 4. 文件放置策略

### 可放 U 盘

- 安装器。
- 启动器。
- UI 面板。
- 示例项目。
- 低敏模板。
- 开源依赖安装脚本。
- `.env.example`。
- 操作教程。

### 不放 U 盘

- `.env.local`。
- 真实 API Key。
- 付费行业包完整源文件。
- 核心 prompt 全量。
- 客户聊天记录。
- 客户图片。
- 服务器 SSH key。
- 支付密钥。

## 5. 授权模型

建议第一版：

```text
用户账号
-> 激活码
-> 机器指纹
-> 套餐权限
-> 行业包权限
-> 用量额度
```

授权校验：

```text
本地启动校验
云端功能二次校验
行业包下载校验
token gateway 调用校验
```

## 6. 输出水印

输出视频、图片、报告可加入：

- 显性品牌水印。
- 隐形用户标识。
- 文件元数据。
- 生成任务 ID。

用途：

- 防倒卖。
- 追踪泄露。
- 区分用户产物。

## 7. 产品话术

不要对客户强调“防破解”。

对外表达：

```text
账号授权、云端更新、行业模板持续升级、保护用户数据安全。
```

内部表达：

```text
防复制、防盗版、防 API 滥用、防模板泄露。
```

## 8. 后续增强

- 客户端代码签名。
- 自动更新器。
- 本地模板加密。
- 离线试用时间限制。
- 远程吊销。
- 异常设备提醒。
- 行业包版本控制。
- 云端审计日志。
